# special-chars — Special Characteristics Management (CC/SC/HIC)

## 行业背景

特殊特性是汽车行业中区分"普通质量问题"和"安全/法规/功能问题"的关键符号系统。IATF 16949 第 8.3.3.3 条（特殊特性）要求组织"识别、记录、控制"所有特殊特性，且必须"贯穿整个供应链"。这不是可选的标注偏好——它决定了供应商选择、过程控制等级、SPC 能力要求、围堵强度、客户升级路径和 PPAP 递交等级。

三类特殊特性的定义边界：

- **CC（关键特性，▽符号）**：任何偏离直接影响安全（导致人员伤害）或违反政府法规（导致不合规）。过程能力要求：Ppk ≥ 1.67。控制等级：防错（消除级）优先，退而求其次用 100% 自动检测，不允许依赖人工检验。
- **SC（重要特性，◇符号）**：任何偏离直接影响产品功能、配合、或客户满意度（非安全）。过程能力要求：Cpk ≥ 1.33。控制等级：SPC + 防错/检测级。
- **HIC（高影响特性）**：部分 OEM（如 Stellantis、GM）使用的额外层级，介于 CC 和 SC 之间。控制要求通常等同 SC 或更严，需查客户 CSR。

特殊特性的识别链必须保持一致性闭环：图纸（标注符号）→ DFMEA（失效模式标注 CC/SC）→ PFMEA（特性列与分类列）→ 控制计划（控制方法按分类差异化）→ SOP/作业指导书（操作员可知可识别）→ 检验规范（检验频次和方法按要求匹配）。IATF 审核中，这条链的任何断裂都属于严重不符合项（8.3.3.3 是最常被开不符合项的条款之一）。

## S1 | 发散探索 — 多角度触发器

Agent 从以下问题中选 3-4 个向用户提问。

1. **产品特殊特性识别**："这个问题涉及的产品，图纸上标注了哪些特殊特性？CC 几个、SC 几个？分别是什么特性？这些特性在图纸上用的是什么符号（▽/◇）？你有没有特殊特性清单文件？"

2. **失效关联检查**："这次的失效模式，是否涉及任何一个图纸上标注的 CC 或 SC？如果不确定——DFMEA 或 PFMEA 里，这个失效模式对应行有没有标注 CC/SC 分类？"

3. **控制方法核实**："涉及 CC/SC 的工序，当前控制计划上写的控制方法是什么——SPC？Poka-Yoke？100% 自动检测？还是人工抽检？这个控制方法和 IATF 要求的最低控制等级是否匹配？"

4. **供应链贯穿**："如果这个特性是从供应商来的（比如来料属性），供应商的特殊特性清单上有没有标这个特性？供应商的 PFMEA 有没有对应行？供应商的过程能力数据（Ppk/Cpk）你有没有？"

5. **SPC 数据状态**："涉及 SC 的特性，Cpk 是否 ≥ 1.33？涉及 CC 的特性，Ppk 是否 ≥ 1.67？最近一次能力报告是什么时候？有没有趋势图？"

## S2 | 第一性原理收敛

追问链 —— 沿识别链逐环节检查闭合性。

**Layer 1 — 图纸到 PFMEA 一致**
- "图纸上标的 CC/SC 符号，在 PFMEA 特性列里是否完全一致？有没有图纸上标了但 PFMEA 没列的特性？反过来呢——PFMEA 列了但图纸没标的特性？"
- "如果图纸有修订历史——最新版图纸的特殊特性标注和上一个版本有什么变化？PFMEA 的版本跟上了图纸修订吗？"

**Layer 2 — PFMEA 到控制计划一致**
- "PFMEA 里这个失效模式行的分类是 CC/SC——对应的控制计划里，控制方法是否和其他 SC/CC 一致？有没有一个 CC 的控制方法低于 SC 的情况？"
- "控制计划里有没有单独的特殊特性控制栏？还是混在常态特性里？操作员能不能直接区分哪些是 CC 特性必须零容忍？"

**Layer 3 — 控制执行追溯**
- "涉及 CC 的特性，防错装置是预防性还是检测性？验证频次是多少？最近一次防错验证记录在哪里？"
- "如果 CC 特性依赖 SPC 监控（而非防错），这样的设计决策是怎么通过的？谁批准的？有没有书面理由？"

**Layer 4 — 供应链深度**
- "供应商交付的产品如果有 CC/SC，你的来料检验是按什么频次检验这些特性的？逐批还是抽样？如果是抽样，AQL 是多少？供应商的过程能力报告你多久要一次？"
- "供应商如果变更了涉及 CC/SC 的工艺，他们通知你的触发机制是什么？在变更批准之前，你是否要求他们重新提交过程能力数据？"

## S3 | 反事实挑战

1. "假设这个 SC 特性当前的 Cpk 报告是 1.35（刚过线），但报告中数据点里有 3 个接近规格限——如果下次能力验证 Cpk 掉到 1.30 以下，控制计划里的反应计划是什么？这个反应计划是否已经验证过？"
2. "如果有人说'这个特性虽然是 SC 但我们一直用人工抽检控制没问题'——你需要什么证据来确认'没问题'不是'还没出问题'？人工抽检的检出率是多少？漏检率是多少？有没有 Red Rabbit 数据？"
3. "如果供应商提供的这个 CC 特性的过程能力报告是假的或过时的——你怎么发现？你的供应商过程审核有没有覆盖这个具体工序的能力验证？最近一次是什么时候？"

## S4 | 答案评分

评分维度：**识别一致性**、**控制充分性**、**链条可追溯性**、**供应链贯穿度**

| 等级 | 标准 |
|------|------|
| Insufficient | 不知道产品是否有 CC/SC；失效模式是否涉及特殊特性不清楚；图纸/PFMEA/CP 三者有任何一个环节断裂；控制方法低于 IATF 最低要求但无法解释；无 SPC 数据 |
| Basic | 特殊特性清单存在；图纸-PFMEA-CP-SOP 链完整；控制方法匹配特性等级；SPC 数据存在且达标（Cpk≥1.33 / Ppk≥1.67）；供应商特殊特性已传递 |
| Excellent | 特殊特性识别链在每个环节都有交叉索引（图纸号-PFMEA行-CP工序号-SOP编号）；控制方法有验证证据（防错验证记录/Red Rabbit 数据）；SPC 数据有趋势分析和预测；供应商端有双向特殊特性管理机制（含过程审核覆盖）；特性变更触发联动更新机制有效 |

## 纠偏原则 / 禁止项

| 禁止 | 替代方向 |
|------|---------|
| 图纸标了 CC/SC 但 PFMEA 对应行没标 | 逐行核对，确保图纸-PFMEA-CP-SOP 每个环节符号一致 |
| CC 特性用人工抽检控制而无书面理由 | CC 必须防错或 100% 自动检测；如果确实做不到，需书面技术理由 + 客户 CSR 批准 + 增强验证 |
| 把 SC 当 CC 或把 CC 当 SC | 按定义重新分类——CC 仅安全和法规，不要过度分类；也不要为逃避 CC 要求故意降级 |
| 供应商端特殊特性不传递 | 采购合同中明确特殊特性条款；PPAP 提交级别按特殊特性数量提高；来料检验覆盖特殊特性 |
| SPC 数据只看达标不看趋势 | 趋势比单点更重要——连续 7 点同一侧即使 Cpk ≥ 1.33 也需调查 |
| 特殊特性只检查一次 | 年度 PPAP / PFMEA 评审必须重新验证特殊特性当前控制有效性；供应商年度审核必须覆盖特殊特性控制 |
