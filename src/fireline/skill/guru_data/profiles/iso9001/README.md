# ISO 9001:2015 Profile for 8d-guru

将此 profile 叠加到 8d-guru 基础 skill 上，Agent 会融合 ISO 9001:2015 质量管理体系要求。

## 适用场景

- 你所在的组织通过了 ISO 9001:2015 认证（或正在准备认证）
- 客户投诉/不符合项调查需要满足 ISO 9001 第 10.2 条的证据标准
- 内审或外审时，审核员要求看 8D 报告的体系合规性
- 你需要确保纠正措施闭环到了管理评审和文件化信息更新

## 核心差异（vs 基础 8d-guru）

| 领域 | 基础 8d-guru | + ISO 9001 Profile |
|------|------------|-------------------|
| 问题定性 | 客户投诉驱动 | 区分客户投诉 vs 内审发现 vs 过程绩效异常（各有不同处理路径） |
| 根因分析 | TRC + MRC + Escape Point | 附加：根因与过程相互作用的关联分析（ISO 9001:2015 强调"过程方法"） |
| 围堵 | 五个风险窗口 | 附加：不符合输出控制（8.7）的五个处置路径——返工/返修/让步/报废/降级——每个需记录和责任人审批 |
| 纠正措施 | PCA 评选矩阵 | 附加：措施与"风险和机遇"的匹配性审查（6.1），措施不能引入新风险 |
| 验证 | 统计证据 | 附加：监测资源（量具/设备）有效性确认（7.1.5），如果量具过期→验证数据无效 |
| 文件化信息 | deliverable 模板 | 附加：文件化信息的创建/更新/控制要求（7.5），每个交付物有保留期限和访问控制标注 |
| 人员能力 | 交叉职能团队 | 附加：团队成员的 competence 证据要求（7.2），关键岗位需列出培训/技能/经验证据 |
| 预防 | Yokoten + FMEA 更新 | 附加：管理评审输入要求（9.3），预防措施需进入管理评审议程 |
| 结案 | 团队签核 | 附加：内部审核追踪（9.2），8D 闭环应作为下次内审的核查项 |

## ISO 9001:2015 核心哲学

2015 版和 2008 版的关键区别：

1. **基于风险的思维**：不再是"出了问题再改"，而是"每个动作先评估风险"。8D 的纠正措施也应如此——不能解决一个问题引入三个新问题。
2. **过程方法**：不再按部门看问题，而是按输入→活动→输出的过程链看问题。根因分析需覆盖过程接口的断裂。
3. **领导力参与**：管理评审不只是签个字，管理层需要对 8D 暴露的系统性问题做决策。
4. **文件化信息而不是文件/记录**：强调信息对决策有用，而不是为了审计归档。
