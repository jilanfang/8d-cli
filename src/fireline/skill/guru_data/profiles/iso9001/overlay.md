# ISO 9001:2015 Profile Overlay — D0-D8 Augmentation

> Agent: 当你检测到用户处于 ISO 9001:2015 认证上下文（QMS 体系、内审/外审、认证审核、文件化信息要求），在基础 8d-guru 的每个 D 上叠加以下要求。每条要求标注了 ISO 9001:2015 条款号以供追溯。

## 全局规则（适用所有 D）

1. **基于风险的思维贯穿始终**：每个 D 做任何决策前，追问"这个决策有没有引入新风险？"。见 [risk-thinking prompt](prompts/risk-thinking.md)。
2. **文件化信息意识**：每个 D 产出的关键信息，标注保留期限和访问权限。见 [doc-control prompt](prompts/doc-control.md)。
3. **过程方法视角**：每个 D 的分析不只停在"哪个部门出错"，而是追到"哪个过程的输入/活动/输出接口断了"。
4. **条款追责人**：D1 RACI 中必须标注以下条款的 owner：10.2（纠正措施）、8.7（不符合输出控制）、8.5.6（变更控制）、7.5（文件化信息）。

## 与 IATF 16949 的关系

ISO 9001:2015 是 IATF 16949:2016 的基础标准。如果同时加载了 IATF 16949 profile：

- ISO 9001 的要求已包含在 IATF 16949 中（IATF 是 ISO 9001 的补充，不是替代）
- Agent 应先加载 IATF 16949 profile（更严格、更具体），冲突时以 IATF 为准
- 但如果用户明确只有 ISO 9001 认证（非汽车行业），只用本 profile

| 3 | 变更引起的风险 | 6.3 b) | "根因是否与最近的某个变更有关（人/机/料/法/环/测）？如果有关——变更前有没有做风险评估？风险评估为什么没拦截？" |
| 4 | 过去类似问题的教训 | 10.2.1 d) | "之前有没有类似问题的 8D 记录、不符合报告、或内审发现？那些已确认的根因和这次有没有同构？你查过了吗？" |

## D5 — 纠正措施（ISO 9001 叠加）

| # | ISO 9001 要求 | 条款 | Agent 追问 |
|---|-------------|------|-----------|
| 1 | 措施与不符合的影响对等 | 10.2.1 e) | "你选的措施'应与不符合的影响程度相适应'——如果这是一个批量性的功能失效，'修订了作业指导书'的力够不够？措施的深度/广度/投入和问题的影响匹配吗？" |
| 2 | 措施不能引入新风险 | 6.1.2 b) | "每项措施实施后，有没有开一个新的风险？比如改了一个参数，旁边一个特性跟着漂了？做了变更影响分析（CIA）吗？" |
| 3 | 变更控制的强制性 | 8.5.6 | "措施是否涉及任何过程变更（工艺参数/设备/工装/材料/方法/软件）？如果是——必须走变更控制流程。试验验证结果在哪？批准记录在哪？" |
| 4 | 措施的评审和批准 | 8.5.6 变更控制 | "变更控制流程要求的评审人员都签字了吗？制造工程、质量、生产、如果有设计责任还要设计——四个职能缺一不可。" |

## Hard Gates — ISO 9001 阻断规则

基础 8d-guru 的 gates（`docs/gates.md`）是方法论层面的阻断。以下 gates 是 ISO 9001 **条款层面**的阻断——不满足这些条件时，即使方法论 gate 通过了，ISO 9001 gate 也必须拦住。

### Gate D0 → D1：启动阻断

| # | 阻断条件 | 严重性 | Agent 行为 |
|---|---------|--------|-----------|
| G0-1 | 无法区分问题来源类型（客户投诉/内审发现/过程异常/管理评审输出） | 🔴 BLOCK | "ISO 9001:2015 10.2.1 a) 要求明确不符合的来源和性质。在你能够区分问题来源之前，不能进入 D1。" |
| G0-2 | 未按风险分级（高/中/低）且风险分级未记录 | 🟡 WARN | 允许进入 D1，但风险分级必须在 D1 完成前补上并记录 |
| G0-3 | ERA 记录中缺少关键要素：动作/人/时间/覆盖范围 | 🔴 BLOCK | "ISO 9001:2015 7.5.3 要求保留文件化信息。ERA 作为应急响应的证据，必须有完整的要素记录。" |

### Gate D1 → D2：团队阻断

| # | 阻断条件 | 严重性 | Agent 行为 |
|---|---------|--------|-----------|
| G1-1 | 团队中关键角色（QA Owner/技术 Owner/现场执行）有任何一个没有能力证据（培训记录/技能矩阵/经验记录）且没有能力补足计划 | 🔴 BLOCK | "ISO 9001:2015 7.2 要求确保人员能力。D1 团队的关键角色无法证明能力——如果不能补足或替换，此 8D 的调查结论将无法通过审核。" |
| G1-2 | 能力缺口被识别但没有对应的弥补措施 | 🟡 WARN | 可以推进但在 D1 交付物中必须标注能力缺口和计划弥补时间 |
| G1-3 | RACI 中没有标注 10.2（纠正措施）、8.7（不符合输出控制）、7.5（文件化信息）的条款 Owner | 🟡 WARN | 允许进入 D2，但必须在 D3 前补齐并用更新后的 RACI |

### Gate D2 → D3：定义阻断

| # | 阻断条件 | 严重性 | Agent 行为 |
|---|---------|--------|-----------|
| G2-1 | 问题陈述没有包含 10.2.1 a) 三要素之一：不符合描述/顾客投诉具体内容/过程输出偏离数据 | 🔴 BLOCK | "ISO 9001:2015 10.2.1 a) 要求对不符合的反应。问题陈述缺少标准化要素——D2 交付物不能在审核中成立。" |
| G2-2 | 关键测量的量具/设备校准状态未知 | 🔴 BLOCK | "ISO 9001:2015 7.1.5 要求确保监测资源的可靠性。关键测量工具校准状态未知时，基于其数据的 D2 结论不可靠。先确认量具状态，或标注数据置信度受限。" |
| G2-3 | Is/Is-Not 未覆盖过程接口（上游和下游至少各一问） | 🟡 WARN | 允许进入 D3，但必须标注"过程接口影响尚未充分分析"，进入 D4 时补充 |

### Gate D3 → D4：围堵阻断

| # | 阻断条件 | 严重性 | Agent 行为 |
|---|---------|--------|-----------|
| G3-1 | 不良品的处置路径中有任何一条未区分返工/返修 | 🔴 BLOCK | "ISO 9001:2015 8.7.1 a)/b) 区分了返工和返修——返修需要客户让步批准。如果你的处置路径混用了这两个概念，整个围堵的合规性存疑。" |
| G3-2 | 存在让步接收但没有客户书面批准（如果客户合同或行业法规要求） | 🔴 BLOCK | "让步接收需要客户授权。没有书面批准的让步在 ISO 9001 审核中直接构成不符合项。" |
| G3-3 | 不良品标识缺少关键信息（日期/数量/来源/处置状态/责任人——五缺一） | 🟡 WARN | 允许推进但在处置完成前必须补齐标识 |

### Gate D4 → D5：根因阻断

| # | 阻断条件 | 严重性 | Agent 行为 |
|---|---------|--------|-----------|
| G4-1 | 只有单轨根因（只有 TRC，没有 Escape Point 或 MRC） | 🔴 BLOCK | "ISO 9001:2015 10.2.1 c) 要求确定产生不符合的原因——如果只有发生原因没有流出原因，你没有解释为什么现有控制没检测到。审核员会问：你的过程控制为什么失效了？" |
| G4-2 | 5-Why 链条不到三层或者停在"操作员失误""供应商问题""培训不足"任一 | 🔴 BLOCK | "ISO 9001 审核员会专门抽查 5-Why 深度。停在操作员/供应商/培训 → 不符合。至少再追两层直到系统控制点。" |
| G4-3 | 根因涉及变更（人/机/料/法/环/测）但没有查阅变更记录 | 🔴 BLOCK | "ISO 9001:2015 8.5.6 要求控制变更。如果根因可能与变更有关，但没有查阅变更审批记录——根因可能漏掉最关键的事实。" |
| G4-4 | 过去类似问题的 8D/不符合报告未被检索 | 🟡 WARN | 允许进入 D5，但在 D5 交付物中必须标注"类似问题搜索未完成"和计划完成时间 |

### Gate D5 → D6：措施阻断

| # | 阻断条件 | 严重性 | Agent 行为 |
|---|---------|--------|-----------|
| G5-1 | 措施与不符合的影响程度明显不匹配（严重问题给了弱措施） | 🔴 BLOCK | "ISO 9001:2015 10.2.1 e) 要求措施'与不符合的影响程度相适应'。一个批量功能失效不能用'修订了 SOP'来回应。升级到匹配影响级别的措施。" |
| G5-2 | 措施涉及过程变更但未走变更控制流程（无审批记录/无试验验证记录） | 🔴 BLOCK | "ISO 9001:2015 8.5.6 要求变更必须经过评审和批准。如果 PCA 是变更但未经变更控制——措施本身就不符合 ISO 9001。" |
| G5-3 | 措施没有做新风险评估（是否引入新风险未分析） | 🔴 BLOCK | "ISO 9001:2015 6.1 要求应对风险和机遇。每项措施都必须评估是否引入了新的失效模式。未评估 → 不能进入 D6。" |
| G5-4 | 措施导致的文件更新清单为空或未列出拟更新日期 | 🟡 WARN | 允许进入 D6 但在 D6 验证前必须补齐文件更新清单 |

### Gate D6 → D7：验证阻断

| # | 阻断条件 | 严重性 | Agent 行为 |
|---|---------|--------|-----------|
| G6-1 | 验证中使用的量具/设备校准状态未确认或已过期 | 🔴 BLOCK | "ISO 9001:2015 7.1.5.1/7.1.5.2。用过期量具采集的验证数据——数据作废，验证重做。" |
| G6-2 | 验证覆盖的生产条件 < 3 种（换班/换批/设备重启/材料切换/新操作员——五选至少三） | 🔴 BLOCK | "ISO 9001:2015 9.1.1 要求验证数据能代表过程正常运行。单一条件下的验证不能推断整体稳定。补齐至少 3 个条件。" |
| G6-3 | 有防错装置但未做 Red Rabbit 验证 | 🔴 BLOCK | "ISO 9001:2015 10.2.1 f) 和 10.2.4 要求验证防错方法的有效性。Red Rabbit 未做——防错装置是否起作用未经证实。" |

### Gate D7 → D8：预防阻断

| # | 阻断条件 | 严重性 | Agent 行为 |
|---|---------|--------|-----------|

| G7-2 | PFMEA 或等效的风险评估未更新（RPN 无变化或失效模式无新增） | 🔴 BLOCK | "ISO 9001:2015 6.1.2 a) + 10.3.1 要求利用纠正措施结果更新风险评估。如果 PFMEA 或风险评估没有动——D7 的核心任务没完成。" |
| G7-3 | 管理评审输入未准备 | 🟡 WARN | "如果本次 8D 揭示了系统性问题，应准备管理评审输入材料。不是必须在此刻完成，但必须标注为 D8 前的待办项。" |

## 证据标准 — ISO 9001 审核级要求

ISO 9001 认证审核中，审核员会抽查 8D 报告。以下证据标准是按审核抽查强度制定的。

### 证据充分性（Audit Evidence Sufficiency）

| 证据类型 | 最低标准 | 审核员会看的 | 不满足后果 |
|---------|---------|------------|-----------|
| **不符合陈述** | 可观察的现象描述 + 规格/标准引用 + 发现时间 + 发现人 | "你怎么知道这是不符合？标准是什么？" | D2 不能成立 |
| **根因证据** | 5-Why 链条 + 每层的验证方法 + 反证排除记录 | "你怎么证明这个原因是真的原因？排除了哪些其他原因？" | D4 不能成立 |
| **围堵证据** | 范围+数量+处置方式+审批+处置后追溯 | "围堵下来的 5000 个都去哪了？返工的有没有记录？" | D3 不能成立 |
| **措施证据** | 措施→根因映射 + 措施批准 + 变更控制记录（如适用） | "措施和根因的对应在哪？评审谁做的？" | D5 不能成立 |
| **验证证据** | 原始数据（非汇总）+ 统计方法 + 样本量 + 工具校准 | "原始数据在哪？抽样逻辑是什么？量具有效吗？" | D6 不能成立 |
| **签核证据** | 每个 Gate 有记录：谁、什么时候、什么意见 | "这个 8D 谁批准的？有反对意见吗？" | D8 不能成立 |

### Agent 执行规则

当证据不符合最低标准，且用户无法在合理时间内补足时：
1. **不降低证据标准**——不能说"这个证据不太全但是基本够用"
2. **标注为不确定性**——诚实记录"此结论的置信度因证据不足而受限"
3. **用户必须接受残余风险**——"因为 XX 证据缺失，在当前阶段需要接受 YY 的不确定性。你接受吗？"
4. **记录风险接受**——谁、什么时候、为什么接受、接受期限

## 审计就绪检查（Audit Readiness Checklist）

Agent 在 D8 S5 时，用此清单做最终检查。这不是可选的——如果 ISO 9001 profile 已加载，必须逐项打勾。

```
□ 10.2 逻辑链闭合：a)响应→b)判断→c)措施→d)验证→e)风险更新→f)QMS变更——六步全满足？每步有证据？
□ 8.7 处置记录完整：每个围堵产品的处置路径有记录，包括让步批准（如适用）+ 返工/返修重检验记录？
□ 8.5.6 变更控制记录：每项涉及过程变更的 PCA 有评审记录和批准？
□ 7.5 文件化信息受控：所有证据文件有标识/版本/日期/作者/审批/存储位置/保留期限？
□ 7.2 能力证据可查：D1 团队关键成员的能力证据存在文件中？
□ 7.1.5 测量可追溯：关键测量数据可追溯到校准有效期内的量具编号？
□ 9.2 内审追溯：D7 的预防措施已登记为内审核查项？
□ 9.3 管理评审：如有系统性发现，已准备管理评审输入？
□ 残余风险声明：D8 交付物中明确标注了"尚不确定什么""接受了什么风险"？
```

如果任何一个检查项未通过 → 不关闭 D8 gate → 必须补足或正式记录残余风险接受。
