# doc-control — 文件化信息控制（ISO 9001:2015 第 7.5 条）

## 行业背景

ISO 9001:2015 把旧版的"文件控制"（4.2.3）和"记录控制"（4.2.4）合并为第 7.5 条"文件化信息"。这不是文字游戏——2015 版想强调的是"信息对决策和运行有用"，而不只是为了审计归档。

2015 版对文件化信息的要求分布在三个子条款：
- 7.5.1 — 总则：QMS 应包含标准要求的文件化信息 + 组织认为必要的文件化信息
- 7.5.2 — 创建和更新：文件化信息在创建和更新时应确保适当的标识、格式、评审和批准
- 7.5.3 — 文件化信息的控制：应控制文件化信息的可获得性（可获取）、充分性（够详细）、适宜性（适用当前情况）、保护（防止泄密和丢失）

对 8D 的影响：
- 8D 全过程的文件化信息不是一份报告，而是一个证据包——包含调查记录/试验报告/评审会议纪要/签核记录
- 每个文件应有标识、版本、作者、日期、审批
- 文件应当可以被需要它的人获取（不只是存在 QA 的硬盘里）
- 外审时，审核员不只查报告——他们会抽查中间证据，比如"5-Why 分析的工作表在哪？试验数据在哪？"

## S1 | 发散探索 — 多角度触发器

Agent 从以下问题中选 3-4 个向用户提问。

1. **文件化信息清单**："这个 8D 的过程中产生了哪些文件化信息？不只是 8D 报告——调查过程中的测量数据、5-Why 工作纸、团队会议记录、试验报告、客户邮件——列一个完整的清单。"

2. **文件标识和版本**："每份文件有没有：a) 唯一标识（文件编号/文件名）b) 版本号 c) 发布日期 d) 作者 e) 审批人？如果一份试验报告连日期和作者都没有——审核员会直接判定它为无效证据。"

3. **文件存储和访问**："这些文件存在哪里？共享盘？QA 的本地电脑？邮件附件？如果审核员要查——你能在多少时间内找齐所有文件？如果 QA 离职了——这些文件还有别人能找到吗？"

4. **文件保护**："关键的文件有没有备份？试验数据有没有只在一台设备里未导出？如果那台设备坏了数据就丢了——数据还没成为文件化信息。"

5. **文件的适宜性维护**："D7 更新了 SOP/控制计划/FMEA 之后——旧版本有没有被移除使用现场？有没有替代/作废标记？现场操作员用的是新版吗？"

## S2 | 第一性原理收敛

追问链 —— 验证文件化信息控制的闭合性。

**Layer 1 — 创建质量的符合性**
- "8D 报告本身——有没有标题、日期、版本、作者、审批？报告里引用的数据，能不能追溯到原始记录（测量报告编号、试验编号、会议编号）？"
- "如果引用了一份外部文件（供应商的试验报告、客户的检测标准）——这份外部文件也受控吗？有版本记录吗？客户标准过期了没？"

**Layer 2 — 分发和可获得性**
- "谁需要看到这个 8D 报告？只有 QA 和客户？制造工程要不要看？现场班组长要不要看？管理层需要看摘要吗？——分发清单是什么？"
- "现场的 SOP 改完以后——有没有发到每个相关工位？每个工位一份还是电子屏显示？如果操作员要查 SOP——他能 10 秒内找到正确的版本吗？"

**Layer 3 — 保留和处置**
- "这个 8D 的文件化信息——保留多长时间？三年？产品生命周期+1年？客户合同有没有特殊要求？"
- "保留期满后怎么处置？是否有销毁记录？——文件化信息的生命周期不只是创建和更新，还包括到期处置。"

## S3 | 反事实挑战

1. "假设三年后你不在公司了，外审员要查这份 8D 的 5-Why 分析证据——接手的人能不能从文件系统里找到这份文件？文件名叫什么？在哪个文件夹下？"
2. "如果操作员说'我按照新 SOP 做的'——但他的工位上的 SOP 是旧版——你怎么证明'不是你文件没分发到位而是他拿错了'？分发记录在哪？"
3. "假设 FBI（或你的关键客户）要求你提供这个 8D 的所有证据——你能不能 24 小时内给出一个完整的证据包，每个文件都有标识/版本/日期/作者/审批？"

## S4 | 答案评分

评分维度：**文件清单完整性**、**标识与版本规范**、**可获得性**、**保护与保留**

| 等级 | 标准 |
|------|------|
| Insufficient | 没有文件清单；关键证据无日期/作者/版本；文件散落在个人电脑或邮件中；没有分发/保留记录 |
| Basic | 文件清单覆盖主要交付物；每个文件有基本标识（至少名称+日期）；存储在共享位置；有保留期限 |
| Excellent | 完整证据包覆盖调查全过程的文件化信息；每个文件有唯一标识/版本/日期/作者/审批；文件可被需要它的人在 1 分钟内获取；关键文件有备份；外部引用文件有版本跟踪；分发清单明确到角色；旧版文件有替代/作废标记；有保留+处置计划 |

## 纠偏原则 / 禁止项

| 禁止 | 替代方向 |
|------|---------|
| "文件都有，在电脑里" | 列文件清单，逐项确认标识/版本/存储位置 |
| 数据只在设备里没导出 | 数据导出并存档，确保在设备故障或更换后仍可获取 |
| 把邮件附件当受控文件 | 邮件中的文件如果有版本——正式版本应该在文件控制系统中，邮件只是分发渠道 |
| 旧版本和新版本共存于现场 | 文件分发必须有回收旧版的机制；电子分发应有版本强制刷新 |
| 没有保留期限 | 按产品生命周期+客户合同+法规要求确定保留期限并记录 |
