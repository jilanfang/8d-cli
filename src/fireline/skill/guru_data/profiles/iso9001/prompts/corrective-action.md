# corrective-action — 纠正措施（ISO 9001:2015 第 10.2 条）

## 行业背景

ISO 9001:2015 第 10.2 条（不符合和纠正措施）是整个 8D 流程的"体系级锚点"。第 10.2.1 条列出了当不符合发生时组织应采取的措施序列，这个序列和 8D 天然对应：

a) 对不符合做出响应（→ D0 ERA）...并控制纠正不符合（→ D3 围堵）
b) 评价是否需要采取措施消除产生不符合的原因（→ D0 启动判断）
c) 实施所需的措施（→ D4 根因 → D5 纠正措施）
d) 评审所采取纠正措施的有效性（→ D6 验证）
e) 必要时更新策划期间确定的风险和机遇（→ D7 预防 → risk-thinking prompt）
f) 必要时变更质量管理体系（→ D7 预防 → 管理评审输入）

本条是认证审核中抽查频次最高的条款之一。审核员不会只看你的 8D 报告格式，而是看这条 10.2 的逻辑链是否闭合：问题来了、措施定了、措施验证了、风险更新了——四个环不能断。

## S1 | 发散探索 — 多角度触发器

Agent 从以下问题中选 3-4 个向用户提问。

1. **不符合的体系影响评估**："这个不符合如果在外审中被发现——审核员会怎么判定？轻微不符合还是一般不符合还是严重不符合？如果严重——需要额外措施吗？从审核的角度反过来看你的 8D。"  

2. **措施严格度与影响的对等性**："你打算实施的措施，和这个问题的严重性匹配吗？如果是批量性、功能级、有客户投诉的问题——措施不能是'修订了作业指导书'。影响越大，措施应该越硬——防错、自动化、消除级。"

3. **类似问题的搜索和批量处理**："这个问题是孤立的吗？你有没有在不符合数据库里搜索过类似问题？如果过去 12 个月发生过类似问题——不应该分开处理，该合并成系统性问题统一处理。"

4. **措施的优先排序**："你有几项措施在做？如果资源不够同时推——按什么排序优先？风险最高的优先？还是最容易的先推？还是客户投诉的先推？说出你的排序逻辑。"

5. **纠正 vs 纠正措施的区别**："'换掉这批不良品'是纠正——消除了不符合本身。'修改了工艺参数防止再产生不良'是纠正措施——消除了不符合的原因。你选的措施是纠正还是纠正措施？D5 应该只有纠正措施——纠正在 D3 已经做了。"

## S2 | 第一性原理收敛

追问链 —— 验证 10.2 逻辑链闭合。

**Layer 1 — 措施与根因的对应性**
- "每个 PCA 是否对应一个具体的根因——TRC 还是 MRC 还是 Escape Point？如果你说不清某个 PCA 对应哪个根因——这个 PCA 可能是惯性操作（'出了问题我们一般就这么改'），而不是针对性措施。"
- "MRC 的措施和 TRC 的措施分开列——用管理措施去补技术根因是错的。比如根因是设备精度不足，措施不能是'加强点检频次'——那是检测层，精度不足是技术层。"

**Layer 2 — 措施进入体系的深度**
- "这个措施只是解决了这个产品的这个问题——还是改善了过程本身？如果同一台设备用来做其他产品——其他产品也受益吗？如果只改了这个产品的 SOP，没改设备的 PM 计划——这个措施没进体系。"
- "这个措施是否同时更新了相关联的体系文件：控制计划/检验规范/过程流程图？还是只在 8D 报告里写了一行。"

**Layer 3 — 措施的可持久性**
- "三年后——新来的工程师看这个 8D，能理解当时为什么选这个措施吗？有没有记录选项对比和淘汰理由？"
- "如果推动这个措施的关键人离职了——措施还能继续执行吗？还是靠这个人的个人习惯在维持？你要保证措施不依赖特定的人。"

## S3 | 反事实挑战

1. "如果措施实施后，同样的问题又发生了一次——但这次是在另一个产品上——这说明什么？说明你的措施够不够深？还是根因没找对？"
2. "如果客户不接受你的纠正措施方案，要求你改用另一种方案——你的 Plan B 是什么？你现在有 Plan B 吗？"
3. "假设半年后——措施的遵从度下降了（操作员开始走捷径、检验频次被悄悄降低、SPC 控制限没人看）——怎么防止这种情况？"

## S4 | 答案评分

评分维度：**措施-根因对应性**、**措施深度**、**体系整合度**、**可持久性**

| 等级 | 标准 |
|------|------|
| Insufficient | 措施和根因对应关系不明确；纠正和纠正措施混淆；只有"加强培训""加强检验"类的措施；措施力度与问题的严重性不匹配 |
| Basic | 每个 PCA 对应一个具体根因；措施至少有一项达到检测级以上；措施已落实到文件更新（SOP/CP/规范）；关键措施有实施计划和时间线 |
| Excellent | TRC/MRC/Escape Point 各对应独立措施；至少一项关键措施达到消除级；措施同时更新了相关联的体系文件（CP/SOP/检验规范/FMEA）；有措施选项对比和淘汰理由记录；措施不依赖特定人员（嵌入了设备/系统/流程而非人的记忆）；考虑了措施可持久性（定期审计、遵从度监控） |

## 纠偏原则 / 禁止项

| 禁止 | 替代方向 |
|------|---------|
| 纠正（换货）当纠正措施（改工艺） | D3 围堵 + 纠正 → D5 才是纠正措施。不要混 |
| PCA 和根因无对应 | 每个 PCA 必须明确标注对应的 TRC/MRC/Escape Point |
| 措施全在检测级 | 至少探讨过消除级的可行性，如果无法消除，书面记录理由 |
| 措施只有一句话无执行细节 | 每个 PCA 要有 Owner/Deadline/Resources/Verification method 四个要素 |
| 不区分临时/永久措施 | 如果措施是分阶段实施的，明确标注哪些是临时哪些是永久，以及临时转永久的时间线 |
