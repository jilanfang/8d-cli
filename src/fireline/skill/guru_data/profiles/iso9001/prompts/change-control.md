# change-control — 变更控制（ISO 9001:2015 第 8.5.6 条）

## 行业背景

ISO 9001:2015 第 8.5.6 条要求"组织应评审和控制生产和服务提供的变更，以确保持续符合要求"。这个条款在 2015 版中从旧版的过程控制中独立出来，成为了一条单独的、带审核频次要求的条款——意味着审核员会专门抽查变更记录。

与 8D 的直接关联：
- 纠正措施中的任何过程变更（参数/方法/设备/材料/软件/人员）都必须走变更控制，不是"D5 选定了就可以直接改"
- 变更前需要评审和批准（不只是质量部，制造工程/生产/技术都要签）
- 变更后需要验证"变更实现了预期效果"——这个验证结果就是 D6 证据的一部分
- 变更对"产品或服务的符合性没有不利影响"——这个就是 risk-thinking prompt 要解决的问题

## S1 | 发散探索 — 多角度触发器

Agent 从以下问题中选 3-4 个向用户提问。

1. **变更类型识别**："每项 PCA 属于什么类型的变更？工艺参数？设备/工装？材料/物料？方法/SOP？软件/MES程序？人员/组织？不同类型有不同的变更评审要求。"

2. **变更范围确认**："这个变更只影响一个产品/一个工序，还是多条产线/多个产品共用？变更范围决定了审批人和验证覆盖范围。"

3. **变更前数据基线**："变更前的数据基线在哪——变更前的参数值是什么？变更前的不良率是多少？变更前的 Cpk 是多少？没有基线，就没法判断变更有没有效果。"

4. **临时变更的管控**："这个变更是不是永久变更？如果是临时变更（过渡方案），临时期限是什么？到期怎么切换回正式方案？临时变更比永久变更更需要严格管控——因为没人会追过期日期。"

5. **紧急变更的补办**："如果变更已经实施了（紧急处置），有没有事后补走变更评审？紧急变更是审核员最爱查的点——他们会问'紧急变更的审批是不是事后才签的'。"

## S2 | 第一性原理收敛

追问链 —— 逐层审查变更控制的闭合性。

**Layer 1 — 变更评审的完整性**
- "变更评审单上，4M 四个维度（人/机/料/法）的评审意见都完整吗？制造工程说 OK，质量部说 OK，设备部说 OK，生产说 OK——缺哪一个？"
- "如果变更涉及材料/物料变更，供应商有没有被通知？他们的 PPAP/首件需要重新提交吗？"

**Layer 2 — 变更验证的充分性**
- "变更验证的样本量是多少？工艺参数变更——验证运行了几个班次？材料变更——验证了几批料？如果只跑了一批/一班——够不够证明稳定性？"
- "验证的数据收集了哪些指标？只看产品符合性——还是也看了过程稳定性、节拍影响、工装磨损、能耗变化？"

**Layer 3 — 变更生效的控制**
- "变更的文件什么时候生效？SOP 改好日期是什么？现场用的是新版还是旧版？操作员怎么知道改用新方法了？——假设一个夜班操作员，他怎么知道今天开始做法变了？"
- "旧版文件/旧工装/旧物料怎么处置？有没有旧版文件仍然在现场的情况？旧工装有没有物理撤出？旧物料有没有区分标识？"

## S3 | 反事实挑战

1. "如果这个变更的验证数据是用一个特别好的班次/特别有经验的老师傅做出来的——下个月换了一班新手，验证结论还成立吗？"
2. "如果这个变更需要半年后才能看出来负面影响（比如加速了工装磨损、材料疲劳）——你现在用什么方法规避这个长期风险？"
3. "假设审核员随机抽查这个变更记录——你认为他会挑出什么问题？变更前没做风险评估？变更后验证数据不够？还是变更批准的人没覆盖所有相关职能？"

## S4 | 答案评分

评分维度：**变更识别完整性**、**评审多职能覆盖**、**验证设计充分性**、**旧状态清理**

| 等级 | 标准 |
|------|------|
| Insufficient | 变更类型未识别；验证只在单一条件下进行；只有一个职能签字；旧版文件/工装处置未说明；临时变更无期限 |
| Basic | 变更类型+范围明确；多职能签字完整；验证覆盖至少 2 个生产条件；旧状态处置有方案 |
| Excellent | 变更影响评估覆盖上下游过程接口；验证覆盖换班/换批/设备重启等边界条件；旧版文件回收+旧工装撤出+旧物料隔离有核查记录；临时变更有到期自动提醒机制；变更后一定周期（如30天）有跟踪验证 |

## 纠偏原则 / 禁止项

| 禁止 | 替代方向 |
|------|---------|
| PCA 实施后补走变更流程 | 变更必须先评审后实施；仅在紧急围堵时可例外但事后必须在规定时间内补办（一般 24-48 小时） |
| 变更评审只有 QA 一个签名 | 按变更类型确定必要的签审职能——缺人就是不符合 |
| 验证只跑一班/一批就宣布成功 | 验证设计需覆盖生产中的正常变异（换班/换批/设备重启/材料切换） |
| 旧版 SOP 还存在现场而新 SOP 已发布 | 旧版文件撤收必须和文件发布同步——用分发-回收记录证明 |
| 临时变更不设过期日期 | 任何临时变更必须有截止日期和到期后恢复或转正的计划 |
