# 基于风险的思维 — ISO 9001:2015 核心方法论

> **目的**：为 Agent 提供一个结构化框架，用于在每个 D 步骤中嵌入风险思维。这不是一个独立的步骤，而是贯穿 8D 全过程的思考方式。

---

## 什么是"基于风险的思维"

ISO 9001:2015 将风险思维从旧版的可选"预防措施"升级为贯穿 QMS 每个环节的强制要求。标准正文中没有"预防措施"这个独立条款（旧版 8.5.3 已删除），因为"风险"的思考方式已经融入了第 4、5、6、7、8、9、10 章的所有相关条款。

核心公式：

```
风险 = 不确定性对预期结果的影响
```

在 8D 的语境里，有三种风险需要同时管理：

1. **原问题的风险**：这个不符合如果复发，后果是什么？（→ 确定 8D 深度和措施的力度）
2. **措施的风险**：纠正措施本身会引入什么新风险？（→ risk-thinking prompt 的核心）
3. **不行动的风险**：如果不采取措施，或者只采取弱措施，复发的后果是什么？（→ 帮助判断措施是否足够）

---

## 风险判断六维度框架

Agent 在帮助用户评估风险时，使用此六维度框架。不要只问"这个风险大不大"——那等于没问。

### 1. 严重度（Severity）

**追问模板**：
- "如果这个失效发生在客户那里——下游会发生什么？产品功能丧失？安全风险？还是只是外观瑕疵？"
- "按你的行业标准，这个严重度等级是多少？（1-10）"

**判定指引**：
- 安全/法规相关 → 9-10 → 措施必须到消除级
- 主要功能丧失 → 7-8 → 措施至少检测级以上
- 次要功能/外观 → 4-6 → 检测级可接受
- 几乎无影响 → 1-3 → 纠正即可，不一定需要永久措施

### 2. 发生率（Occurrence）

**追问模板**：
- "这个问题多久发生一次？每批都有？每月一次？还是第一次？"
- "在当前控制下，这个失效模式的发生概率是多少？"

**判定指引**：
- 每批/每天 → 8-10 → 过程能力不足
- 每周/每月 → 5-7 → 特殊原因+部分系统原因
- 每年/首次 → 1-4 → 可能是特殊原因，但首次 ≠ 最后一次

### 3. 探测度（Detection）

**追问模板**：
- "当前的检测手段能多大概率发现这个失效？100% 自动检？抽检？还是靠客户发现？"
- "如果客户比你先发现——探测度就是最低的。"

**判定指引**：
- 靠客户发现 → 9-10 → 防错优先级最高
- 人工抽检 → 6-8 → 有漏检风险
- 100% 自动检测 → 3-5 → 合理，但设备本身也有可靠性问题
- Poka-Yoke（不可能发生/不可能流出）→ 1-2 → 理想状态

### 4. 影响范围（Scope）

**追问模板**：
- "这个问题影响了多少产品/客户/产线/订单？不是一个数字，是四个维度的覆盖范围。"
- "上游和下游有没有关联影响？供应商的来料影响了几批？客户的这批次用了多少台产品？"

### 5. 时间紧迫性（Urgency）

**追问模板**：
- "客户给了多少时间？有没有合同里规定了 8D 提交时限？"
- "生产线停了吗？在等你的围堵结论才能复工？"

### 6. 检测能力可靠性（Measurement Confidence）

**追问模板**：
- "你判断合格/不合格的检测手段——可靠到什么程度？量具校准了吗？GR&R 做了吗？"
- "如果有两个检验员分别判断——结论一致吗？"

## 风险应对的五种策略

ISO 9001:2015 第 6.1.2 条要求组织策划应对风险的措施。五种策略可组合使用：

| 策略 | ISO 9001 对应 | 在 8D 中的应用 |
|------|-------------|--------------|
| **规避**（Avoid） | 消除风险源 → 不做这个动作 | 设计变更让这个失效模式不可能出现 |
| **消除**（Eliminate） | 消除风险源 → Poka-Yoke | 物理上让失效不可能发生 |
| **降低**（Reduce） | 降低发生的可能性或后果 | 加强控制、增加检测、改进过程能力 |
| **转移**（Transfer） | 分担风险 → 如供应商承担 | 要求供应商 8D + 提高来料检验等级 |
| **接受**（Accept） | 保留风险——在风险低或控制成本过高时 | 低 RPN 项，有书面接受记录 |

## 在各 D 步骤中的风险追问速查

| D 步骤 | 风险追问重点 |
|--------|------------|
| D0 | 这个不符合的影响范围多大？安全/功能/法规/数量/客户——按哪个维度判断严重度？ |
| D1 | 团队成员有没有能力风险（关键技能缺失？）外部资源能兜底吗？ |
| D2 | 测量数据的可靠性有风险吗（量具过期/GR&R 不合格？）NTF 风险处理了吗？ |
| D3 | 围堵措施有漏网风险吗（抽样而非 100%？检出率验证了吗？）围堵产品处置的新风险（让步品混入？） |
| D4 | 候选根因中哪个的风险最高？如果放弃这条线索，漏掉真因的风险是什么？ |
| D5 | 每项 PCA 引入了新风险了吗？措施力度和风险的严重度匹配吗？ |
| D6 | 验证条件的局限性会漏掉什么风险？验证覆盖了生产中的正常变异吗？ |
| D7 | 类似产品/过程没有排查的风险有多大？FMEA 没更新的风险是什么？ |
| D8 | 不确定性的诚实声明——我们还不能确定什么？不声明就是把不确定当确定的风险 |

---

## 风险接受的三条铁律

1. **不能靠"感觉"接受风险**：风险接受必须有理由（成本/技术不可行/概率极低）、有数据支撑、有管理层审批。
2. **风险接受必须限时**：今天接受风险不意味着永远接受。设定复审周期（如 6 个月或下一次管理评审时）。
3. **风险接受必须可追踪**：在 8D 中记录——我们接受了什么风险、为什么接受、谁批准的、什么时候复审。
