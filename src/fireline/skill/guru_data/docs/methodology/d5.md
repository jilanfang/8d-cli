# D5 — 纠正措施

## 行业最佳实践

D5 在 AIAG/VDA 和 VDA 4 中均被拆为两个独立步骤：先选择措施（Choose），再验证措施在实验条件下有效（Verify）。这与 D6 的"投产后验证"有本质区别 — D5 是实验室/小试/试产阶段的确认，D6 是量产条件下的持续证明。

行业标准要求三项强制动作：

1. **每项纠正措施必须对应一个已确认的根因**。根因有三种：发生根因（TRC，为什么发生）、流出根因（MRC，为什么没拦住）、系统根因（Escape Point，为什么控制点失守）。每个措施必须说清它针对哪一类根因，不允许"一个措施解决所有问题"。
2. **Poka-Yoke 分级评估**：每项措施必须标记防错等级。三级体系 — 消除（从物理上杜绝错误可能，最高级）> 检测（错误发生时立即捕获，中级）> 减轻（降低错误影响，最低级）。没有 Poka-Yoke 评估的 PCA 选择是不完整的。
3. **同步更新 PFMEA 和控制计划**。措施一旦选定，对应的 PFMEA RPN 值和控制计划控制方法必须同步修订，不能在 D7 才想起来更新文件。

## 探索方向矩阵

|探索方向|示例问题|
|------|------|
|PCA 筛选方法|用时间/成本/效果/难度四维评分矩阵筛选候选措施时，效果权重是否高于成本权重？如果两项措施效果等同怎么办？|
|风险预判|这项措施本身可能引入什么新失效模式？有没有在 PFMEA 里预判过？措施的副作用谁来跟踪？|
|Poka-Yoke 设计|当前措施属于消除/检测/减轻哪一级？能不能从检测级升级到消除级？升级的技术可行性和成本是多少？|
|验证方案设计|D5 验证（试验条件下）和 D6 验证（量产条件下）的判定标准有什么不同？样本量怎么定？验证周期多长？|
|责任与时间锚定|每项措施的责任人是不是具体个人？完成时间有没有考虑采购周期、工装加工周期、软件变更窗口？|
|临时措施退出路线|D3 ICA 什么时候可以撤？撤出条件和 D5 PCA 的生效条件之间怎么衔接？撤出顺序是什么？|
|控制点修复层级|这个措施修的是哪个控制点？是只修了检测层还是修了设计层？如果只修检测层，为什么不能往前推到设计层？|

## 第一性原理锚点

D5 的本质问题是：**我们有没有把根因翻译成了能阻断因果链的具体动作？**

追问到这个层才算到位：

- 这条措施在因果链上切断了哪一环？是切断了发生路径（TRC）、流出路径（Escape Point）、还是两条都切了？
- 如果这条措施失效了，问题会以什么形式复现？我们怎么知道它失效了？
- Poka-Yoke 等级为什么不选更高一级？是技术做不到、成本不划算、还是没想过？

## 常见跳步模式和阻止策略

**跳步 1：不筛选直接定措施**
表现：想到一个措施就写上去，没有列出候选措施做比较。
阻止策略：要求每条根因至少列出 2 个候选措施，用四维矩阵打分后再选定。如果只有一个候选措施，必须说明"为什么没有替代方案"。

**跳步 2：措施和根因不对应**
表现：根因是"回流温度曲线无控制限"，措施是"加强培训"。
阻止策略：每条措施写完后，反向追问"这条措施解决了哪个根因的哪条因果链？"如果答不上来，重写。

**跳步 3：Poka-Yoke 不评估**
表现：所有措施默认是"加强检验"，没有任何防错设计。
阻止策略：每项措施强制标注 Poka-Yoke 等级。如果全是"检测"级，要求对 Top 3 高风险措施给出升级到"消除"级的方案，即使最终不做也要记录为什么不做。

**跳步 4：PFMEA/控制计划更新被推迟**
表现：D5 选了措施但说"文件等 D7 再更新"。
阻止策略：文件更新不是 D7 的专属任务。D5 选定的每项措施，必须在 D5 阶段同步标记需要更新哪些 PFMEA 行、哪些控制计划控制方法。D7 是做横向展开，不是做首次更新。

## 收敛过早的警告信号

- 每条根因只有一个措施，没有替代方案讨论。
- 所有措施的 Poka-Yoke 等级全是"检测"，没有一条讨论过"能不能消除"。
- PCA 选择没有用任何矩阵或打分方法，纯凭经验直觉。
- 没有明确区分 D5 验证（试验条件）和 D6 验证（量产条件）的判定标准。
- PFMEA RPN 更新没做，或只改了发生度没改探测度。
- ICA 退出计划缺席：没说什么时候、在什么条件下撤 D3 的临时围堵。

## Excellent 级别标杆

- 每条根因对应一个经过四维矩阵（时间/成本/效果/难度）评分的 PCA 选择，附替代方案和落选理由。
- 所有 PCA 标注 Poka-Yoke 等级，至少一项关键措施达到"消除"级。
- D5 验证方案与 D6 验证方案分开定义，判定标准可量化。
- PFMEA 和控制计划已在 D5 阶段同步标记修订内容，责任人明确。
- ICA 退出条件和 PCA 生效条件有明确的交接时间线，不会出现"围堵撤了、PCA 还没上"的空窗。
- 每项措施附"措施失效检测机制"：如果 PCA 没起作用，我们怎么知道、多久能知道。
- 如果所有措施的 Poka-Yoke 等级都在检测级以下，已用 TRIZ 8 原理（见 `docs/patterns/triz-8-principles.md`）挑战了"为什么不能升级到消除级"。
