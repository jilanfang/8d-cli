# D7 — 预防系统

## 行业最佳实践

D7 在丰田体系中称为"Yokoten"（横展），字面意思是"横向展开到所有地方"。它不是写一份经验总结就完事，而是一组必须执行到位的系统加固动作。D7 最容易做成面子工程 — 文件更新了但不生效，横向检查了但没落地。

行业标准要求四项并发动作：

1. **横向部署（Yokoten）**：在相似产品、工艺、产线、供应商、客户应用中逐一检查同样的根因和 Escape Point 是否存在。范围至少包括：同平台产品、同工艺产线、同物料族、同供应商的其他料号。每项检查必须有结论 — 有风险就开整改，没风险就记录排除理由。
2. **四类文件同步更新**：FMEA、控制计划、SOP、图纸/规格书四者必须同步修订。只更新其中一两份是伪 D7 — 操作员看 SOP、质量看控制计划、工程师看 FMEA，任何一份未更新就会产生信息断层。更新顺序：先改 FMEA（定风险），再改控制计划（定方法），再改 SOP 和图纸（落地）。
3. **经验教训入库**：建立可检索的 Lessons Learned 数据库，不是丢进共享文件夹的 PDF。每条经验至少标注：失效模式、根因类型、产品/工艺族、Poka-Yoke 等级、有效措施摘要。将来同类问题发生时，工程师能搜到前人怎么解决的。
4. **系统缺口溯源**：追问当初是什么决策、假设、压力让这个控制点从一开始就没有设置或设置不当？这不是追责，是暴露系统性盲区。例如：当初为什么没有在这个工序设置防错？是项目周期压太紧、经验库里没有类似失效、还是做 PFMEA 时默认这个失效"不可能发生"？

## 探索方向矩阵

|探索方向|示例问题|
|------|------|
|横向部署范围|同类产品范围怎么定 — 按产品平台、按工艺路径、还是按物料 BOM 交集？同供应商的其他料号要不要查？已经停产的产品要不要查？|
|文件同步机制|四份文件（FMEA/CP/SOP/图纸）的更新有没有顺序？如果先改了 SOP 但 PFMEA 还没改会怎样？怎么确保四份文件的实际执行人拿到了新版？|
|经验教训可检索性|这条经验以后怎么被搜到？用哪些标签 — 失效模式、根因类型、产品族、还是工序代码？谁来维护这个库？旧的、过时的经验怎么清理？|
|组织学习|当初什么决策/假设/压力造成了这个系统漏洞？团队有没有在复盘这个？如果同一个假设备下次还出现，我们怎么能更早识破？|
|审计验证|D7 的动作怎么验证它真的落地了 — 是看文件版本号、走现场审计、还是看后续几个月的数据？谁来审、什么时候审？|
|供应商横向展开|如果根因在供应商端，供应商的横向展开怎么跟踪？谁去验证供应商真的改了？供应商改完后我们这边什么文件也要跟着改？|
|预防性维护衔接|这个案例的失效模式能不能纳入 PM 计划？设备维护周期要不要调？备品备件策略要不要改？|

## 第一性原理锚点

D7 的本质问题是：**我们有没有把这个案例的痛苦，转化为组织中每个相关节点都变强一点的永久改变？**

追问到这个层才算到位：

- 如果同一个工程师离职了，新来的人能不能通过 Lessons Learned 数据库在 10 分钟内找到这个案例和相关措施？如果不能，说明知识还在人脑里，没进系统。
- 横向检查的结果是列表还是行动？列了一堆产品名说"没风险"不算 D7，得说明"为什么没风险"的证据是什么。
- 四份文件改完后，现场的人实际在用的是新版本还是旧版本？文件版本号更新不等于行为变更。

## 常见跳步模式和阻止策略

**跳步 1：只修当前产品，不做横向检查**
表现：D7 输出是"经检查无同类产品"或根本没提同类检查。
阻止策略：要求列出至少 3 类横展对象，每类至少一个具体料号，逐一标注"有风险/无风险及证据"。

**跳步 2：只更新一份文件**
表现：只改了 SOP，FMEA 和控制计划没动。
阻止策略：D7 模板要求四份文件各自标注状态（已更新/不需更新/N/A），每份附版本号和责任人。

**跳步 3：经验只存不进**
表现：写了一份 8D 报告 PDF 存进共享盘，没有结构化入库。
阻止策略：经验录入至少包含 5 个结构化字段：失效模式、根因分类、产品/工艺族、Poka-Yoke 等级、有效措施标签。

**跳步 4：不追问系统缺口**
表现：找到技术根因就停止，不问"为什么当初会这样"。
阻止策略：强制回答一个问题："当初什么决策/假设/压力导致这个控制点缺失或不足？"

## 收敛过早的警告信号

- 横向检查结论只有"无同类产品"，没有列出具体排查范围和排除依据。
- 只改了一份文件，其他文件的状态是"待更新"。
- 经验总结写成"大家以后注意"或"加强管理"，无任何可检索字段。
- 没有追问"当初为什么没有想到"。
- 没有审计计划 — 不知道谁来验证 D7 动作落地。
- 供应商端的横向展开没有追踪，因为"供应商不在我们体系内"。

## Excellent 级别标杆

- Yokoten 范围覆盖同产品平台、同工艺路径、同物料族、同客户应用四维，每维有具体排查结果和排除依据。
- PFMEA/控制计划/SOP/图纸四份文件同步更新，版本号和责任人可追溯，现场审计确认新版在执行。
- 经验存入结构化 Lessons Learned 数据库，至少 5 个可检索标签，经验内容包含"如果重来一次会怎么做"。
- 系统缺口分析回答了"当初什么决策/假设/压力导致漏洞"，并提出了防止同类系统盲区的改进建议（如 PFMEA 评审清单更新、项目阶段性评审增加防错检查点）。
- 审计计划已定义：谁在多长时间后复审、复审哪些指标、复审不通过怎么处理。
