# D1 — 团队成立

## 行业最佳实践

IATF 16949 和 Ford 8D 手册对 D1 的要求非常明确：8D 不是一个人写的报告，是一群有相关知识和权限的人共同完成的调查。团队缺角，调查必然有盲区。

行业核心规则三条：

1. **跨职能最小集合**：至少覆盖 production + quality + engineering 三个职能。production 知道实际怎么做出来的，quality 知道检测和控制点在哪里，engineering 知道设计和工艺本应怎么工作。缺任何一个，调查会在对应维度出现空白。
2. **具体个人，不是部门。** 写"质量部"等于没写——没人知道该找谁、出了问题该追谁。每个角色必须落到具体姓名，附带联系方式。这不是官僚主义，这是可追溯性：三个月后回头查这个 8D，需要能找到当时每一个角色是谁。
3. **RACI 明确**：每个角色必须明确是 Responsible（干活的人）、Accountable（最终负责的人）、Consulted（需要征求意见的人）还是 Informed（需要知道进展的人）。Accountable 角色只能有一个人——Owner 就是这个 A。

D1 做弱了的后果：调查中途发现"这个角度没人看"，回头补人，时间窗口已经过了——证据没了、记忆模糊了、客户耐心耗尽了。

## 探索方向矩阵

| 方向 | 示例问题 |
|------|---------|
| Owner 确认 | 谁对这个 8D 的过程和结果最终负责？这个人有没有权限调动所需资源？ |
| 技术覆盖 | 产品设计、制造工艺、测试验证三个技术维度是否都有对应人员？ |
| 现场执行 | 谁能实际进入产线/仓库/客户现场收集一手证据？不是"可以找人帮忙"而是"自己就能去"。 |
| 客户接口 | 谁对外统一沟通？不同角色给客户不同说法会导致信任崩塌——是否只有一个口径？ |
| 证据管理 | 谁负责证据的收集、编号、保存？D4 分析时找不到 D2 采集的样本是最常见的低级失误。 |
| 权限检查 | Owner 有没有权限要求其他部门配合？如果 Owner 是工程师但需要停线，跨部门权限够不够？ |
| 外部资源 | 需要供应商参与吗？需要客户配合吗？需要第三方实验室吗？ |

## 第一性原理锚点

D1 的追问应该停在这一层：**"要完整描述和解决这个问题，信息必须从哪些视角来？每个视角有没有一个能独立判断、能直接操作的人？"**

追问路径：

- 从问题倒推视角：这个失效涉及设计吗？→ 需要设计工程师。涉及制造吗？→ 需要工艺/生产。涉及检测吗？→ 需要质量/测试。画完视角地图再看有没有人对应。
- 每个角色问三层：这个人能不能直接看到一手信息（不是听别人转述）？这个人能不能独立做技术判断（不是只能传话）？这个人有没有时间投入（不是在忙别的事"顺便"参与）？
- RACI 交叉检查：同一个角色不能同时是 R 和 A——干活的人和负责的人分开。A 只能有一个，避免"集体负责等于没人负责"。

## 常见跳步模式和阻止策略

**跳步模式 1：一个人包办所有角色。** 用户说"我来当 Owner，技术也是我，客户接口也是我"。识别方式：角色清单里同一个人名出现三次以上。阻止方式："如果所有信息都从你一个人来，D2 的问题定义就没有交叉验证。至少要有一个能独立质疑你判断的人。"

**跳步模式 2：角色写成部门名。** 用户说"质量部负责"。识别方式：RACI 表里出现部门名称而非人名。阻止方式："质量部里有五个人，三个月后谁来回答这个 8D 的问题？我们需要具体姓名和联系方式。"

**跳步模式 3：缺少现场执行角色。** 团队里全是坐在办公室的人，没有人能直接去产线或客户现场。识别方式：所有角色都是"经理""工程师"头衔，但没人负责实际采样/测量/拍照。阻止方式："D2 需要去现场看实际问题，团队里有谁能直接进产线？"

## 收敛过早的警告信号

- 团队少于 3 人，或虽然人多但职能重复（三个质量工程师，没有生产和工程）。
- Owner 是级别最低的团队成员，没有跨部门协调的权限。
- RACI 里 A 角色有多个，或 R 角色全部空缺（没人干活）。
- 缺少联系方式——不只是缺手机号，连"通过什么渠道找这个人"都不清楚。
- 客户接口角色空缺，或客户接口和技术 Owner 是同一个而这个人说话不谨慎。

## Excellent 级别标杆

- RACI 矩阵完整，每个角色的 R/A/C/I 有且仅有一个标签，A 只有一个人。
- 团队覆盖 production + quality + engineering 三个职能，且有现场执行和证据管理。
- 每个角色标注了姓名、联系方式、可投入时间百分比。
- Owner 有明确的授权来源：是质量经理指定的、是客户要求的、还是问题严重度自动触发的。
- 团队名单已经和客户确认过（如果涉及客户沟通），客户知道找谁。
- 如果团队缺少某个职能，有书面说明原因和替代方案——不是遗漏，是主动判断。
