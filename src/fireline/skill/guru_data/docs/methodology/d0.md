# D0 — 准备与应急

## 行业最佳实践

D0 不是 8D 的正式第一步，而是"决定是否启动 8D"的前置闸门。ASQ 和 Ford 8D 手册明确将 D0 定位为 Emergency Response Action（ERA）阶段——问题暴露后的第一反应窗口。

行业标准做法是三层判断：

1. **严重度闸门**：按 safety → function → regulation → volume → customer impact 逐层判级。安全风险直接触发最高级别响应，不受其他因素稀释。
2. **ERA 执行**：在根因完全未知的条件下，先做三件事——隔离受影响产品、通知受影响方、保护现场证据。ERA 不问"为什么"，只问"现在要做什么来止损"。
3. **启动判断**：严重度达到阈值或客户正式投诉时，启动完整 8D。低于阈值的做简化记录和围堵即可。

D0 做弱了，整个 8D 会在错误级别上运行：该进 8D 的没进，后续系统性根因永远挖不出来；不该进 8D 的进去了，资源浪费在低风险问题上。

**D0 的 ERA 和 D3 的围堵不是一回事。** ERA 是问题刚暴露时、信息极少条件下的应急动作；D3 围堵是 D2 问题边界明确后、针对已知风险窗口的系统封堵。ERA 快但粗，D3 围堵慢但准。两者不能互相替代：做了 ERA 不等于 D3 可以跳过；D3 也不能因为是"正式围堵"就否定 ERA 的价值。

## 探索方向矩阵

| 方向 | 示例问题 |
|------|---------|
| 现象确认 | 客户实际观察到什么？失效模式是否可复现？有没有照片/视频/实物样本？ |
| 发现链追溯 | 谁第一个发现的？通过什么渠道（投诉/退货/内部检测/第三方）？发现和上报之间延迟了多久？ |
| 严重度判级 | 涉及安全吗？功能完全丧失还是降级？影响了多少客户/产品？有没有法规风险？ |
| 紧急处置盘点 | 已经做了什么？谁做的？什么时候做的？措施是否覆盖所有已知受影响位置？ |
| 证据保护 | 失效件是否已隔离保存？现场数据（参数记录/日志/环境数据）是否已抓取？ |
| 升级判断 | 需要通知哪些层级？客户是否需要 24 小时内回应？是否需要启动召回评估？ |
| 启动决策 | 是否满足完整 8D 的触发条件？如果不启动，替代处理方案是什么？谁来批准这个决策？ |

## 第一性原理锚点

D0 的追问应该停在这一层：**"这个问题在物理世界里实际发生了什么——不是客户描述了什么，不是谁推测了什么，而是可观察、可测量的事件序列是什么。"**

具体追问路径：

- 把客户语言翻译成物理语言：客户说"USB 没输出"，翻译成"5V 输出引脚在负载 500mA 条件下测得电压 0V"。
- 把时间线拉出来：什么时候发生、什么时候发现、什么时候上报，三者之间的延迟就是信息衰减的时间窗口。
- 把"严重"翻译成可排序的维度：安全 > 功能 > 法规 > 数量 > 客户感受。一次只用一个维度判级，不要混在一起说"综合考虑比较严重"。

追到这里就够了。不要在这个阶段追问"为什么会发生"——那是 D4 的事。

## 常见跳步模式和阻止策略

**跳步模式 1：现象没确认就开始分析根因。** 用户上来就说"USB 无输出是因为来料不良"。这是推断不是现象。识别方式：描述里出现了"因为""所以""可能是"。阻止方式："我们先不急着找原因。先把眼睛看到的东西说清楚——客户看到了什么？"

**跳步模式 2：跳过严重度判级直接成立团队。** 用户说"先找人开会"。识别方式：D0 的三个基本问题（现象/来源/严重度）有一个没回答就开始讨论谁参加。阻止方式："这个问题严重到什么程度？需不需要启动完整 8D？我们先判断清楚再组队。"

**跳步模式 3：把紧急处置写成了纠正措施。** 用户说"已经让供应商换料了"就当问题解决了。识别方式：措施描述里出现了"换""改""调整"但没有区分是临时还是永久。阻止方式："这是紧急处置还是最终措施？如果是临时的，我们后面还要找根因。如果是最终的，根因找到了吗？"

## 收敛过早的警告信号

- 现象描述只有一句话，没有可测量的数值，没有时间点。
- 严重度判断用了"比较""相对""一定程度"这类不可比词汇。
- 紧急处置只有一个笼统动作，没有责任人、没有完成时间。
- 用户开始讨论 D4 的根因方向，但 D0 的三个基础问题还有一个没回答。
- 证据保护被忽略——失效件已经报废或退回供应商，现场数据已覆盖。

如果出现以上任一信号，D0 还没有完成。不要进入 D1。

## Excellent 级别标杆

- 现象描述可以直接作为 engineering 问题的输入：有可测量指标、有时间、有地点、有频次。
- 严重度判级有明确的分层依据，不是"感觉严重"而是"功能完全丧失，影响客户产线停线，按严重度矩阵属 A 级"。
- ERA 的记录包含：做了什么、谁做的、什么时候做的、覆盖了哪个风险窗口、为什么选择这个措施而不是别的。
- 证据保护有清单：哪些证据已保存、保存在哪里、谁负责保管、是否已拍照/记录。
- 启动/不启动决策有书面记录和批准人，不是口头决定。
