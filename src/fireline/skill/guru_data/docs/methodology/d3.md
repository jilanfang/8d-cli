# D3 — 围堵措施

## 行业最佳实践

D3 的任务是：在根因还没找到的情况下，先把已经流出的和可能流出的问题产品控制住，防止损害继续扩大。IATF 16949 和 Ford 8D 手册对此有硬性时间要求：确认问题后 24 小时内必须向客户发出 Quality Alert，说明问题性质和初步围堵状态。

核心框架是**五个风险窗口**的逐窗排查：

1. **客户现场**：已在客户手中的产品——是否有问题、是否已隔离、客户是否已被告知。
2. **在途**：已发货但尚未到达客户的产品——能否拦截、谁能拦截、拦截后如何处理。
3. **成品/半成品库存**：已在仓库但未发货的——是否需要 100% 筛选、筛选方法是否已确认有效。
4. **WIP（在制品）**：产线上正在做的——是否暂停生产、暂停范围多大、已产出但未入库的如何处理。
5. **上游（供应商/来料）**：问题是否可能来自供应商、在途来料是否受影响、是否需要供应商配合筛选。

行业标准还要求：

- **100% 受影响数量识别**：不能只知道"大概多少"，需要逐窗确认具体数量。如果可追溯性不支撑 100% 识别，必须如实记录"仅可追溯到批次级别"，这是一个需要后续系统纠正的管理根因。
- **围堵措施有效性验证**：围堵不是"做了就行"。需要用 DOE 或控制图验证筛选方法确实能把不良品从良品中挑出来。如果筛选方法本身的检出率只有 80%，那 20% 的不良品会继续流到客户手中。
- **围堵失效模式分析**：围堵措施本身可能失效——筛选工具坏了、操作员疲劳漏检、标签贴错导致混料。每个围堵措施应做一次轻量的失效模式预判。

**最重要的认知：围堵不等于永久措施。** D3 是止血带，不是手术。止血带能让你活到手术开始，但一直绑着止血带肢体会坏死。D3 完成后必须进入 D4 找根因和 D5 永久措施。行业里最常见的错误就是在 D3 停下来——"反正筛选了、客户也不投诉了，就这样吧"。

## 探索方向矩阵

| 方向 | 示例问题 |
|------|---------|
| 客户现场围堵 | 是否需要通知客户暂停使用？客户现场有多少受影响产品？如何隔离/退货/现场筛选？ |
| 在途拦截 | 在途产品能否截回？物流节点在哪个位置？谁有权限发起拦截？ |
| 库存筛选 | 仓库里有多少需要检查？筛选方法是否验证过有效性？筛选后的良品标签如何区分？ |
| WIP 控制 | 需要停线吗？停线范围多大？在制品如何处理——全检/暂停/标记？ |
| 上游追溯 | 供应商是否已通知？供应商库存和在途有多少？来料批次追溯精度够不够？ |
| 围堵有效性 | 筛选方法的检出率是多少？有没有做 MSA（量测系统分析）确认筛选工具可靠？ |
| 围堵失效点 | 如果筛选操作员疲劳漏检，第二道防线在哪？如果标签贴错导致混料，怎么发现？ |

## 第一性原理锚点

D3 的追问应该停在这一层：**"问题产品在物理世界的所有可能位置都被封堵了吗——不只是最明显的那个位置，而是从问题发生的原点顺物流链向下游所有节点逐一确认。"**

追问路径：

- 从源头向下游走：从供应商来料 → 原材料仓 → 产线 → 半成品仓 → 成品仓 → 物流 → 客户仓库 → 客户产线/终端。每一个节点问三个问题：这里有没有可能有问题产品？用什么方法确认有没有？如果确认不了（可追溯性不足），保守假设是有还是没有？
- 对每个围堵措施问三层：这个措施能检出问题产品吗（有效性）？这个措施本身可靠吗（可靠性）？如果这个措施失效，怎么发现（失效检测）？
- 时间线压紧：每个围堵动作有没有 deadline？有没有责任人？晚一天意味着什么？

## 常见跳步模式和阻止策略

**跳步模式 1：只围堵客户现场，忽略其他窗口。** 用户说"客户那里的货已经换完了"。识别方式：风险窗口只填了客户现场一栏，其他四栏空白。阻止方式："仓库里的同一批产品怎么办？还在路上的怎么办？供应商那里还有没有同批次的？"

**跳步模式 2：围堵措施没有验证过。** 用户说"我们让检验员 100% 挑一遍"。识别方式：问"这个检验方法的检出率是多少"时回答"应该都能挑出来吧"。阻止方式："我们需要确认筛选方法确实有效。可以先拿 20 件已知不良品和 20 件良品混在一起，看检验员能挑出多少。"

**跳步模式 3：把围堵当成终点。** D3 完成后用户觉得"问题解决了"就不再推进 D4。识别方式：围堵措施描述里用了"以后都这么检""已经加了 100% 全检"。阻止方式："筛选是临时措施，成本高且不能防止问题再发生。我们还得找根因，否则同样的问题还会来。"

**跳步模式 4：没在 24 小时内通知客户。** 识别方式：Quality Alert 发出时间距问题确认时间超过 24 小时。阻止方式："客户有权在 24 小时内知道你的产品可能有问题。即使原因还没找到，围堵状态和风险评估必须先传过去。"

## 收敛过早的警告信号

- 五个风险窗口有超过一个未确认——空白或"不涉及"但没有说明判断依据。
- 围堵措施的筛选方法没做过有效性验证。
- 受影响数量的统计是"估计""大约"，没有追溯到具体批次或序列号。
- 围堵措施没有责任人和 deadline。
- Quality Alert 未在 24 小时内发出，或发出了但内容只有一句"我们在调查"没有实质信息。
- 用户开始表现出"D3 做完就行了"的态度，对 D4 缺乏推进意愿。

## Excellent 级别标杆

- 五个风险窗口逐窗排查，每个窗口有：判断依据（为什么涉及/不涉及）、受影响数量（不是估计，是数字）、具体措施、责任人、完成时间。
- 筛选方法经过有效性验证：有 DOE 或 MSA 数据，知道检出率和误判率。
- Quality Alert 在 24 小时内发给客户，内容包括：问题描述、受影响产品范围、围堵措施状态、下一步计划、客户侧建议。
- 围堵措施的失效模式已识别并有二次防护（如：目视筛选 + 抽检验证）。
- 每个围堵措施都有明确的解除条件和过渡到永久措施的计划——围堵不是永远的。
- 如果可追溯性不足以支持 100% 识别，已如实记录并在 D4 的管理根因分析中列为待纠正项。
