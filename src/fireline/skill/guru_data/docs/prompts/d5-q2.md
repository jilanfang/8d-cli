# d5-q2 — 纠正措施对应流出点（Escape Point）

## 行业背景

IATF 16949 对缺陷预防和探测的要求有明确区分：探测（detection）不能替代预防（prevention），但探测层的修复也是 8D 的必要输出。VDA 第 8 卷指出，Escape Point 的修复措施必须回答一个问题：“下一次同样类型的问题发生时，这个控制点能否保证拦住它？”

AIAG CQI-20 强调 D5 阶段需要两类措施并行：一类针对发生根因（occurrence），一类针对流出根因（escape）。前者防止问题再次发生，后者确保万一发生能被拦住。两类措施互补，不可相互替代。

流出点措施的独特性在于：它不阻止问题产生，而是确保问题到不了客户手里。这类措施最常见的错误是“加一道人工检验”而不是从控制系统层面补齐检测能力。

## S1 | 发散探索 — 多角度触发器

1. D4 确认的 Escape Point 是哪个控制点？这项措施如何让这个控制点重新具备检出能力？
2. 之前这个控制点为什么失效了？标准缺失、标准执行不到位、还是标准本身无法检出这类问题？
3. 如果同类型但不同表现的问题再次发生，现有的控制点能拦住吗？
4. 为什么不在更早的控制点拦截？成本、技术还是组织原因？这个决定是经过权衡还是有路径依赖？
5. 这项措施修复的是探测逻辑（能检出）还是执行纪律（有人认真检）？后者为什么在之前失去了？

## S2 | 第一性原理收敛

从控制系统角度追问流出点措施的有效性：

- **探测机制**：控制点是靠什么探测原理工作的？视觉检查、尺寸量测、功能测试、电气测试、还是其他？这个原理对本次失效模式是否敏感？
- **探测门槛**：控制点的判定标准（如公差范围、测试规格）是否包含了本次问题的失效特征？是标准写对了但没执行，还是标准本身就挡不住？
- **探测时机**：控制点在工序流中的位置是否合理？往前偏移能不能更早检出、降低损失？往后偏移有没有泄漏风险？
- **反馈链路**：检出异常后的处置流程是否有效？是停机、隔离、追溯、还是只记录？流程有没有被绕过？

有效措施的特征：探测原理匹配失效模式 + 判定标准包含失效特征 + 时机合理 + 反馈链路闭环。

## S3 | 反事实挑战

- 如果保留现有控制点不变但把判定标准收紧，能拦住这个问题吗？为什么能或不能？
- 假设三个月后同一条线出了另一个质量问题，不同的失效模式，现在的控制点体系有多大把握能拦住？
- 离开这道人工检验，有没有不受人员注意力影响的方式（传感器、防错装置、自动化检测）？

## S4 | 答案评分

| 维度 | 1分 | 3分 | 5分 |
|------|-----|-----|-----|
| 探测敏感性 | 措施无法说明如何检出失效模式 | 措施可能检出但不保证 | 措施对失效模式有可靠检出能力 |
| 系统鲁棒性 | 纯人工依赖，无防错机制 | 有辅助手段但人工仍是关键 | Poka-Yoke / 自动探测，不受人员影响 |
| 反馈完整性 | 检出后无规定处置流程 | 有流程但依赖人为判断 | 检出→自动停线/隔离→根因反馈闭环 |

三项均 ≥ 3 分才进入下一步。

## 纠偏原则 / 禁止项

- 每个 Escape Point 必须有对应措施。如果某个流出点不需要措施，必须解释为什么（如该控制点在本次问题中未失效）。
- 禁止“加强检验”“加人检查”“提高意识”这类不改变探测逻辑的措施。如果加一个人看能解决问题，那原来那个人为什么没看到？
- 优先选择 Poka-Yoke 方案（消除对人注意力的依赖），其次选自动化探测，最后才接受人工加判定标准。
- 流出点措施不是发生根因措施的替补——两类措施必须同时存在。
- 不能只在出货前加一道全检。Escape Point 修复应该在最合理的位置，不是最靠后的位置。
