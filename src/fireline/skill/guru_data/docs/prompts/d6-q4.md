# d6-q4 — 验证结果与下一步决策

## 行业背景

VDA 第 8 卷指出 D6 验证结果必须包含实际执行的数据和基于接受标准的判定。AIAG CQI-20 强调 D6 是 8D 中的决策点：验证通过→进入 D7（标准化和横向展开），验证不通过→回到 D5（重新选择或加强纠正措施）。

这是 8D 全流程中最重要的反馈回路。很多 8D 到了 D6 就走了个过场——验证数据敷衍，不管结果如何都写“验证通过”，然后直接跳到 D8 签字归档。这种 8D 叫“纸面闭环”。

D6 的数据完整性要求：实际执行方法、实际样本量、实际数据、是否满足接受标准、不满足时的下一步动作——五个要素缺一不可。不能只写“已验证有效”五个字替代所有数据。

鼓励 Red Rabbit 测试：在改进后的过程中故意引入已知不良状态，验证过程是否能检出。Red Rabbit 是一种强验证手段，尤其适用于 Escape Point 修复措施。

## S1 | 发散探索 — 多角度触发器

1. 实际执行的验证方法是否和 D6-q1 里计划的一致？如果有差异，什么原因？影响结论吗？
2. 实际样本量是否达到了 D6-q2 的计划？如果没达到，结论的可信度降低了多少？
3. 实际数据是什么？不要只说结论，把原始数据按关键维度列出来——批次、产线、时间序列。
4. 数据是否满足 D6-q3 设定的所有接受标准？每一项标准逐一对照，不要挑着报。
5. 如果不满足标准，下一步不是写原因解释，而是回到 D5 重新评估措施——你回到 D5 的具体动作是什么？

## S2 | 第一性原理收敛

从数据完整性、统计推断和决策链路角度追问验证结果的可靠性：

- **实际 vs 计划一致性审查**：列表逐项比对 D6-q1/q2/q3 的计划和 D6-q4 的实际执行。每一处偏差都必须解释——偏差是否会影响结论的有效性？
- **原始数据完整性**：验证数据必须有原始记录可查。如果原始数据丢失或不完整，验证结论不可接受。数据至少应包含：样本标识、测量值、测量时间、测量人、判定结果。
- **统计推断 vs 直观感受**：即使所有样本都合格，也需要计算统计置信度。例如 59 件 0 不良在 95% 置信水平下意味着真实不良率可能高达 5%。这个置信区间和接受标准是一致的吗？
- **时间序列分析**：不要只看总计，要按时间顺序画趋势。如果前 N-1 批都 0 不良，最后一批出现不良迹象（哪怕还在规格内但在边界），这是早期预警信号。

**Red Rabbit 测试**：如果有条件，用已知不良品或故意制造不良状态来验证改善后的控制点能否检出。通过标准：Red Rabbit 100% 被检出并触发正确的处置流程（停线/隔离/追溯）。

如果验证结果不满足标准，回到 D5 的路径：
1. 确认原根因是否正确 → 如果不正确，回到 D4
2. 确认措施是否符合发生机制 → 如果不符合，重新设计措施
3. 确认措施执行是否到位 → 如果执行不到位，加强执行纪律
4. 确认验证方法是否合理 → 如果方法不对，调整验证方法但不是调整接受标准

## S3 | 反事实挑战

- 如果这个验证数据给一个独立的第三方评审员看，不做任何解释，他能不能独立判断通过还是不通过？
- 如果六个月后同样的问题复发，回头看今天的验证数据，最可能是在哪一步自欺欺人？
- 验证结果的信心来源是什么？是大样本量+一致数据+足够时间，还是“全都合格所以我信了”？

## S4 | 答案评分

| 维度 | 1分 | 3分 | 5分 |
|------|-----|-----|-----|
| 数据完整性 | 只有结论，没有实际数据 | 有数据但缺原始记录 | 原始数据+统计汇总+时间趋势完整 |
| 一致性 | 实际执行和计划有重大偏差未解释 | 偏差已解释但影响未评估 | 偏差已解释且影响已评估 |
| 决策正确性 | 无论数据如何都写“通过” | 判定基本合理但缺置信分析 | 判定有统计和工程双重依据 |

三项均 ≥ 3 分才进入 D7。

## 纠偏原则 / 禁止项

- **强制拒绝以下结论**：
  - “已验证有效”（没有数据）
  - “验证合格”（没有对照标准逐一判定）
  - “问题已解决”（D6 只做效果验证，D7/D8 才是关闭）
- 实际执行和计划有偏差时必须解释，不解释直接退回。
- 原始数据必须可查。没有原始数据的验证结论是无效结论。
- 如果验证不通过，下一步必须写“回到 D5，重新评估 [具体措施]，因为 [具体原因]”，不能只写“继续跟踪”。
- 鼓励 Red Rabbit 测试。如果措施落在 Poka-Yoke 或探测层，Red Rabbit 是检验真伪的最强手段。
- D6 通过验证只表示措施在验证条件下生效，不代表生产条件下的长期可靠性——后者由 D7 的持续监控保障。
- PCA 验证未通过前，D3 的 ICA（临时遏制措施）必须继续有效执行。D6 通过后，PCA 替代 ICA 不是 ICA 自然失效——需要正式撤除 ICA 的决策。
