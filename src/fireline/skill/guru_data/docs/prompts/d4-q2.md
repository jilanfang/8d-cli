# d4-q2 -- 候选原因(6M / Fishbone)

## 行业背景

Fishbone 是 ASQ CQE 必修工具。MAHLE 要求在 D4 用 6M 系统列举候选原因，每个原因必须有支持证据**和**反驳证据。只看支持证据 = 确认偏误，这是 D4 最致命的陷阱。BSH 审核特别看:如果只有证据支持没有反驳，审核员会问"你怎么证明你没有偏见?"

## S1 | 发散探索 -- 多角度触发器

1. 人(Man):操作员当时掌握什么信息?SOP 和实际操作之间有没有差距?疲劳/轮班/技能矩阵有没有关联?
2. 机(Machine):关键参数在规格内吗?CM/CPK 趋势有没有漂移?工装治具有无磨损、松动?
3. 料(Material):批次间有无差异图谱?供应商有无变更?存储条件有无异常?
4. 法(Method):SOP 是否覆盖了失效路径?控制计划有这个参数吗?变更有没有走 ECN?
5. 环(Environment):温湿度/ESD/振动有无波动?有没有季节性/时间段模式?
6. 测(Measurement):量具在有效期内吗?GR&R 合格吗?同一批样品不同人/不同工具测得一致吗?

**初始化贝叶斯概率表**：S1 完成后，读 `docs/patterns/bayesian-hypothesis.md`。为每个候选原因分配初始概率（基于 D2 Is/Is-Not + 时间模式证据）。概率总和 ≤ 100%，保留 5-15% 给"其他未知原因"。每个假设必须写一个"最可能推翻它的证据"。

## S2 | 第一性原理收敛

**物理链条**:对每个候选原因追问"这个 M 的问题，通过什么物理/化学机制导致失效?"答不出来的，可能不直接相关。

**证据质量分层**:支持证据是直接观察还是推理?是定量数据还是定性判断?是同一批次的还是类比外推的?来源独立吗?

**反驳证据检索**:对排第一的候选原因，必须追问:"有没有一个数据点和这个结论矛盾?有没有其他批次满足同样条件但没有失效?"

## S3 | 反事实挑战

1. 如果你排第一的候选原因实际不是根因，排第二、第三的分别是什么?它们被排到后面的理由经得起重审吗?
2. 有没有可能两个或多个候选原因同时作用才触发失效?单因不足以触发，组合才过临界点?

**概率更新**：S3 完成后，更新贝叶斯概率表。反事实通过 → 领先假设概率上升；反事实失败 → 该假设概率下降。领先假设概率变了吗？第二假设呢？系统交互的概率需要调整吗？

## S4 | 答案评分

| 维度 | Insufficient | Basic | Excellent |
|------|-------------|-------|-----------|
| 6M 覆盖 | 缺 2 个以上 M | 6M 全覆盖 | 6M 全覆盖 + 高优 M 有 2-3 个候选 |
| 证据质量 | 只有推断 | 高优候选有证据支持 | 每个候选有证据支持+反驳 |
| 反驳意识 | 无反驳证据 | 排第一的有反驳讨论 | 每个候选有反驳证据或"未找到但需验证" |
| 交互考虑 | 未考虑 | 提到可能性 | 列出组合假设+验证方式 |

D4 铁律:候选原因只有 1 个、或全指向同一个 M、或只有支持无反驳，一律 Insufficient。

## S5 | 输出格式

```json
{
  "candidate_causes": [
    {
      "cause": "可验证的原因描述",
      "category": "人/机/料/法/环/测",
      "priority": "high/medium/low",
      "evidence_for": ["事实+来源"],
      "evidence_against": ["反驳证据 或 '未找到，需通过{方法}验证'"],
      "physical_chain": "原因→失效模式的物理因果链",
      "confidence": 0.0
    }
  ],
  "interaction_hypotheses": ["多因交互假设"],
  "top_candidate": "当前最可能原因",
  "confidence": 0.0
}
```

## 纠偏原则 / 禁止项

| 禁止 | 替代方向 |
|------|---------|
| 只有 1-2 个候选原因 | 回到 6M，每个 M 至少想一个 |
| 只写支持不写反驳 | "什么数据点与这个结论矛盾?" |
| 凭经验直接排到第一 | 按证据权重排序，不是"最常见" |
| "这个 M 不相关"跳过 | 改成"暂未发现关联，需{检查方式}确认" |
| 证据全来自同一来源 | 交叉验证:不同人、不同批次、不同维度 |
