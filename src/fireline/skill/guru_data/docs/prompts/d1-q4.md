# d1-q4 — 现场执行

## 行业背景

现场执行角色是 8D 团队中"去实际发生地看"的人。Gemba（现场）原则是 8D 和精益质量管理的核心信条：二手数据和报告无法替代在现场的直接观察。ASQ 8D 方法论明确要求问题调查必须包括对实际发生地的走访，AIAG CQI-20 把"没有去 Gemba"列为 D4 根因分析质量低的首要原因。现场执行角色不只是"去看看"，而是带着具体的数据收集计划去：什么位置、收集什么、怎么记录、怎么保存。常见的弱执行：派了一个没有技术判断力的人去现场拍了些没重点的照片回来，或者根本没有人去现场、所有人坐在会议室里根据数据和报告分析问题。

## S1 | 发散探索 — 多角度触发器

Agent 从以下问题中选 2-3 个向用户提问。

1. **任务定义**："谁去现场？去哪个现场（产线/仓库/客户现场/供应商现场）？去做什么具体的事？不是'去看看'，而是一个带清单的任务。"
2. **能力要求**："去现场的人有没有能力识别和这个失效相关的关键特征？他知道看什么、量什么、拍什么吗？如果他去了但不知道什么是异常，等于白去。"
3. **时间窗口**："现场现在还能看到什么？失效发生时的现场状态还在吗？产品还在吗？设备设置变了吗？人员换班了吗？如果现场已经变了，这个去现场的人需要找谁重建当时的状态？"
4. **证据收集**："去现场的人需要带什么工具？（相机/量具/标签/样品袋/记号笔）收集什么类型的证据？（照片/视频/测量数据/样品/访谈记录）证据怎么编号和保存？"
5. **访问权限**："这个人能不能进入所有相关现场？客户现场能不能进去？供应商现场能不能进去？需要什么审批或陪同？"

## S2 | 第一性原理收敛

追问链 —— 从表面答案逐层深入：

**Layer 1 — 任务具体化**
- "你说'派人去产线看'—— 具体看什么？看哪个工位？看哪个操作步骤？什么算正常什么算异常？这个判定标准去的人知道吗？"
- "你说'去客户现场'—— 客户同意了吗？什么时候能去？去几个人？需要客户提供什么配合？"

**Layer 2 — 收集计划**
- "去现场的人有没有一份检查表或数据收集计划？还是去了自己看着办？"
- "如果去的人现场发现了一个不在计划里但看起来重要的线索，他能不能偏离计划去追？还是必须报备等批准？"

**Layer 3 — 现场可变性**
- "现场的状态在问题发生和现在之间有没有变过？（设备调过、物料换过、人员换过、方法改过）。如果有，去现场的人应该重点和谁聊？"
- "如果去的人在现场发现'一切正常、看不出问题'，备选行动是什么？是不是需要特定条件复现问题？"

## S3 | 反事实挑战

1. "假设去现场的人回来说'没发现异常'，你相信这个结论吗？你怎么区分'真的没异常'和'有异常但他没能力识别'？"
2. "如果现场最关键的证据（比如失效的那个零件、当时的设备参数、那个班次的操作员）已经不可获取了，替代方案是什么？还是说调查就卡在这里了？"

## S4 | 答案评分

评分维度：**任务明确性**、**能力匹配**、**证据链完整性**

| 等级 | 标准 |
|------|------|
| Insufficient | 没有明确的现场执行人；任务描述模糊（"去看看"）；去的人没有技术判断力；无证据收集计划 |
| Basic | 现场执行人具名；任务有具体检查项；了解证据收集和保存要求；相关现场访问权限已确认 |
| Excellent | 有书面检查表或数据收集计划；去的人有能力识别关键特征并解释判断依据；证据编号和链式保管已规划；现场可变因素已评估（设备/物料/人员/方法四方面） |

## 纠偏原则 / 禁止项

| 禁止 | 替代方向 |
|------|---------|
| "派人去现场看看" 当任务描述 | 必须有检查清单、证据收集计划、判定标准 |
| 派一个不懂产品/工艺的人去现场 | 去现场的人必须有基本技术判断力，或与技术角色一起前往 |
| 依赖已变化的现场做直接观察 | 必须评估现场状态变化，必要时通过访谈重建当时状态 |
| 没有证据链管理计划 | 从收集到保存，证据的完整性是审核必查项 |
| 忽略客户/供应商现场的访问难度 | 提前确认权限和配合，不要"到时候再说" |
