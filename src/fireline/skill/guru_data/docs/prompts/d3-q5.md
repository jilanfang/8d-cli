# d3-q5 — 上游围堵

## 行业背景

IATF 16949 第 8.4 条（外部供方管理）和第 8.7.1 条联合要求：当问题追溯到供应商来料时，围堵范围必须延伸到供应商端——包括供应商的成品库存、在途品、WIP 和原材料。AIAG CQI-20 强调：供应商端围堵的启动速度直接决定了整个围堵链的完备性——供应商多拖一天，就有更多的可疑来料在途中。ASQ 指出，供应商端围堵最大的障碍不是技术上的，而是商务上的：通知供应商问题存在的时机、方式和证据强度，会影响供应商的配合程度。Ford 要求：如果怀疑来料问题，必须在 24 小时内向供应商发出正式的质量问题通知（SCAR 或等同），并明确供应商端的围堵要求。

## S1 | 发散探索 — 多角度触发器

1. **你的来料批次追溯完整吗？** 从客户投诉的问题产品，能一路反向追溯到供应商的哪个批号、哪个发货日期吗？中间经过几道仓库/产线——每个节点的批号转换都有记录吗？
2. **你通知供应商了吗？** 是正式的质量问题通知还是打了个电话？供应商收到通知后回复了吗——回复了什么？有没有要求供应商在 24 小时内给出围堵计划？
3. **供应商那边有多少可疑来料？** 不是问你这里有多少，是问供应商自己还有多少库存、在途、WIP。你要求供应商提供这些数据了吗？供应商给了吗？
4. **供应商端的围堵措施可靠吗？** 你怎么验证供应商真的做了围堵——是供应商自述"我们查过了没问题"，还是你派人去看了，或者看了供应商的围堵记录？供应商的筛选方法和你的筛选方法一致吗？
5. **如果来料问题涉及多个零件号/多个供应商，你全部排查了吗？** 同一个供应商的其他牌号有没有同样的问题？同一个原材料的其他供应商有没有同样的问题？你是不是只查了"出问题的这个料号"就停了？

## S2 | 第一性原理收敛

追问链（对涉及上游的窗口逐条追问）：

- 你说的"已经通知供应商了"，通知里面有没有写清楚问题现象、受影响批次、以及你要求供应商做什么？有没有书面的、带日期的通知记录？供应商有没有书面回复？
- 供应商说"我们查了库存没问题"，他的查法和你内部查问题产品的方法一样吗？如果他的出厂检验项目里根本没有你这次发现的失效模式，他怎么判断"没问题"？
- 供应商端的筛选如果由供应商自己做，你能信任他的筛选结果吗？有没有交叉验证——你从他筛选过的"合格品"里抽一些回来重新筛一遍？
- 如果供应商不配合或者反应很慢，你的备选方案是什么？换一家供应商紧急供货？还是你这里先用自己的来料筛选代替供应商端的围堵？
- 你怎么保证后续来料是干净的？供应商有没有在出货前加了这个失效模式的控制？有没有加在你的来料检验项目里？

## S3 | 反事实挑战

"如果你的上游围堵失败了——供应商说'没问题'但你后来发现他根本没查、或者供应商筛选漏检了继续发货——这批新来的可疑料混进了你的产线，下一批产品继续带着同样的问题出厂、送到客户手里。到那个时候，客户会说'你们不是已经保证这个问题解决了吗'。你的来料检验能在这个失效模式上拦住这批料吗？如果拦不住，你的下一个防线在哪里？"

## S4 | 答案评分

| 维度 | 不通过（0分） | 合格（1分） | 深入（2分） |
| :------ | :--------- | :------- | :------- |
| **供应商通知质量** | "和供应商说了"，没有正式记录 | 有正式通知（SCAR/8D请求）+供应商书面回复 | 有正式通知+供应商回复+围堵要求明确+24小时时效追踪 |
| **验证机制** | 信任供应商自述 | 审核了供应商的围堵记录 | 交叉验证（盲样测试/现场审核/数据对比）+后续来料加严检验 |
| **范围完整性** | 只查了出问题的那一个料号 | 排查了同供应商同原材料的其他料号 | 多料号+多供应商排查+原材料级别追溯+分供方延伸 |

## 纠偏原则 / 禁止项

- **禁止只用"供应商说没问题"作为不围堵的依据。** 供应商的自我声明不能替代验证。必须追问：供应商的检查方法是什么、检查了多少件、检查标准是否覆盖了本次失效模式。如果有条件，做交叉抽样验证。
- **禁止不通知供应商就自己做来料筛选了事。** 供应商有权知道自己的产品出了什么问题。只在自己的来料端加一道筛选不叫上游围堵——围堵要追溯到供应商自己的库存和在途品，防止未受控的可疑品继续流向你或其他客户。
- **禁止不排查多料号/多供应商。** 问题可能不只影响一个料号。同一个供应商用同一批原材料生产的其他零件、同一个原材料的其他供应商——这些都在上游围堵的排查范围内。
- **禁止把"供应商端围堵"和"供应商根因分析"混在一起。** D3 要的是供应商端的隔离和筛选，不是供应商为什么做出不良品。后者是 D4 的工作，留到下一步。
