# d4-q1 -- 根因分析范围

## 行业背景

D4 是 8D 的心脏。ASQ《The Quality Toolbox》:范围划错，后面全是错的。MAHLE 供应商 8D 要求先看全所有方向再做收敛。BSH D4 审核第一项:"是否考虑了多个潜在原因方向"。弱 D4 = 整个 8D 无效。

## S1 | 发散探索 -- 多角度触发器

1. 用 6M 框架逐个想:人/机/料/法/环/测，每个 M 至少列出 1 个可能有关的方向，即使直觉说"不太可能"。
2. 什么时候开始?最近有什么变更?要覆盖:物料批次、设备参数、人员流动、环境季节、工艺调整、供应商切换。
3. 以前有没有类似失效?那次根因是什么?和这次条件完全相同吗?列出差异点。
4. 好的批次和坏的批次之间有什么差异?分层对比:不同产线、班次、模具、时间段。
5. 你已经排除的方向，排除是因为"有反证"还是"没证据所以暂时排除"?这两种要分开写。

## S2 | 第一性原理收敛

**物理机制**:从失效模式反向推，什么物理/化学/材料变化会导致这个现象?能量、应力、相变、扩散--用具体机制说话。不要停留在"焊点开裂"这种结果描述。

**设计假设**:这个工序/产品的原始设计假设了哪些边界条件?正常使用的包络范围是什么?这次失效是否突破了某个设计假设?

**控制回路**:从失效点往回走整条控制链。每个应该拦住这个问题的控制点，当时实际在做什么?应该做什么?差距在参数、标准还是执行?

## S3 | 反事实挑战

1. 如果你最有信心的方向是错的，什么证据能在 24 小时内证明?你现在有这些证据吗?
2. 如果不看"可能性最高"的方向，第二可能的方向被排除的证据够硬吗?

## S4 | 答案评分

| 维度 | Insufficient | Basic | Excellent |
|------|-------------|-------|-----------|
| 覆盖面 | 只列 1-2 个方向 | 6M 全覆盖 | 6M + 变更时间线 + 类似案例对比 |
| 排除依据 | 凭经验排除，无数据 | 排除有理由但无证据 | 有数据支撑排除，区分"无证据"和"有反证" |
| 逻辑链 | 方向选择无因果推演 | 有逻辑推演 | 有物理机制解释 + 考虑交互效应 |

D4 铁律:范围只有一个方向且未经物理机制验证的，一律 Insufficient。

## S5 | 输出格式

```json
{
  "scope_statement": "产品/工序/时间段/失效模式",
  "included_directions": [
    {"direction": "...", "6m_category": "人/机/料/法/环/测", "rationale": "...", "priority": "high/medium/low"}
  ],
  "excluded_directions": [
    {"direction": "...", "exclusion_type": "有反证排除/暂无证据排除", "evidence": "..."}
  ],
  "change_history": ["变更事件+日期"],
  "similar_case": "过去案例及根因(如有)",
  "confidence": 0.0
}
```

## 纠偏原则 / 禁止项

| 禁止 | 替代方向 |
|------|---------|
| "感觉就是这个方向，别的不用看了" | 6M 每个至少说一句:"如果和这个 M 相关，会是什么" |
| "没证据所以排除" | 改成"暂未发现证据，未关闭，需{验证方法}" |
| 跳过变更时间线 | 从发现日往回推 6 个月，列出所有变更 |
| 不同批次数据混在一起 | 分层对比:好 vs 好，坏 vs 坏，再交叉找差异 |
