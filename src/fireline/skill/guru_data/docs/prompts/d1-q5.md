# d1-q5 — 证据管理

## 行业背景

证据管理是 8D 中常被低估但在审核和客户面前最具杀伤力的环节。IATF 16949 第 7.5.3.2.1 条要求组织保留工程变更和批准的记录证据，审核员在核查 8D 时会严格检查证据链的完整性。ASQ 8D 方法论强调：没有编号、没有存储位置、没有保管链的证据不是证据，只是"一堆东西"。证据管理包括：样品（不良品/良品/对比样）、数据记录（测量数据/测试报告/MES 数据/SPC 图表）、影像资料（照片/视频/热成像/X-ray）、文档记录（邮件/会议纪要/判定记录）。常见问题：样品丢了、照片不知道对应哪个产品、测量数据没有记录测试条件、关键邮件被删除。证据链断裂 = 审核不符合项 = 客户信任丧失。

## S1 | 发散探索 — 多角度触发器

Agent 从以下问题中选 2-3 个向用户提问。

1. **责任人**："谁来管理这个 8D 的所有证据？谁来编号？谁来决定什么东西该保存、保存多久？不是'大家各自保存自己的'，而是有一个明确的证据管理角色。"
2. **证据清单**："目前已经收集了哪些证据？（样品/数据/照片/视频/邮件/记录）。还有哪些应该收集但还没收集的证据？列出差距。"
3. **存储与检索**："证据存在哪里？物理样品存哪里？（实验室？仓库？样品柜？编号规则是什么？）电子数据存哪里？（共享盘？SharePoint？邮件？本地电脑？）别人能不能独立找到和访问？"
4. **编号系统**："有没有统一的编号规则？样品编号、数据文件命名、照片命名有没有一致的格式？如果一个陌生人来查这个案例，能不能仅凭编号找到对应的证据？"
5. **保存期限**："这些证据需要保存多久？法定要求是什么（产品生命周期 + 1 年？）？组织的质量记录保存政策是什么？过了保存期后怎么销毁？"

## S2 | 第一性原理收敛

追问链 —— 从表面答案逐层深入：

**Layer 1 — 现状盘点**
- "你现在手上有多少个物理样品？都是什么状态的？（不良品原样/拆解后的零件/良品对比样/复现试验的样品）。每个样品有标签吗？标签上写了什么？"
- "所有照片和视频存在哪里？文件名能不能看出是什么时候拍的、拍的是什么？如果让你现在找一张一周前拍的不良品照片，5 分钟内能找到吗？"

**Layer 2 — 链式保管**
- "样品从现场取回来以后，经过了几道手？每一道手有没有记录（谁/什么时候/为什么转移）？现在样品在谁手上？"
- "如果样品需要送到外部实验室分析，寄送之前怎么记录样品状态？寄送过程怎么保证样品不丢失/不损坏/不被调包？"

**Layer 3 — 证据完整性**
- "有没有证据已经丢失了？（比如不良品被报废了、邮件被删了、监控录像被覆盖了）。丢失了什么？还能不能从其他渠道补救？"
- "如果客户或审核员要求看某个证据，你最担心哪个证据拿不出来？为什么？"

## S3 | 反事实挑战

1. "假设这个 8D 结案一年后，客户回来质疑你们当时的根因结论，要求审查所有原始证据 —— 你们现在的证据管理方式能支撑吗？哪个环节最可能掉链子？"
2. "如果证据管理人离职了，换了一个新的人接手，他能不能仅凭现有的编号和存储系统找到这个案例的所有证据？还是需要靠离职人口的交接？"

## S4 | 答案评分

评分维度：**可追溯性**、**完整性**、**持久性**

| 等级 | 标准 |
|------|------|
| Insufficient | 无证据管理人；证据散布在个人电脑/手机/邮件中；无编号规则；无存储位置；已有证据丢失 |
| Basic | 证据管理人具名；证据集中存放（物理和数字）；有编号规则；主要证据类型已收集；知道保存期限 |
| Excellent | 有证据台账（登记每件证据的类型/编号/位置/收集时间/保管人）；链式保管可追溯；编号规则与公司质量记录管理系统一致；证据缺失项已标注并说明了原因和补救计划 |

## 纠偏原则 / 禁止项

| 禁止 | 替代方向 |
|------|---------|
| 证据管理无责任人，各自保存 | 指定单一证据管理人，所有证据汇集到统一存储 |
| 样品没有标签、照片文件名是一串乱码 | 建立编号规则，标签上至少包含：案例号/日期/产品/内容 |
| 样品/数据存放在个人抽屉或个人电脑 | 必须集中存放并可被授权人员独立访问 |
| 没有证据台账，靠记忆管理 | 建立证据清单，每件证据登记类型/编号/位置/收集信息 |
| 证据缺失但不标注 | 缺失项明确标注，说明原因（丢失/报废/覆盖）和补救计划 |
