# d2-q4 — 影响量化

## 行业背景

8D 的问题陈述必须是 SMART 的，其中 M（Measurable）直接对应影响量化。Ford 8D 手册和 AIAG CQI-20 要求用数据（而非形容词）描述问题规模：数量、缺陷率、成本、客户影响范围。影响量化的目的不只是"汇报严重程度"，更是校准整个调查的资源投入和方法论深度 —— 1 件安全相关产品需要深度工程分析，1000 件外观划痕需要的是围堵和分拣效率。IATF 16949 要求组织在客户投诉处理中量化不良成本和影响。常见问题：数量和成本被低估（只算了客户退回的，没算在途和在制品的），导致 D3 围堵范围定小了，D5 措施力度不够。

## S1 | 发散探索 — 多角度触发器

Agent 从以下问题中选 2-3 个向用户提问。

1. **数量盘查**："现在能确定的问题产品数量是多少？这个数量是精确的还是估算的？从哪些渠道汇总的？（客户退回、内部库存、在制品、在途、客户库存）还有什么渠道没查？"
2. **缺陷率**："问题产品的缺陷率是多少？（不良数/总生产数）。按批次分开算，是不是每个批次缺陷率一样？有没有某个批次缺陷率特别高？"
3. **成本计算**："这个问题到目前为止造成了多少直接成本？（退货物流、报废、返工、分拣人工、停线索赔、紧急出差、加班）。间接成本呢？（客户信任、未来订单风险、品牌声誉）。哪些成本还没算进去？"
4. **客户影响量化**："有几个客户受影响？每个客户影响的程度一样吗？有没有哪个客户受影响特别大？（比如主力客户停线）。客户有没有罚款或索赔条款会被触发？"
5. **风险敞口**："如果不做任何控制，这个问题还可能影响多少产品？你现在的围堵是不是 100% 覆盖了风险敞口？"

## S2 | 第一性原理收敛

追问链 —— 从表面数字逐层深入：

**Layer 1 — 数字来源验证**
- "你说的'影响 500 件'—— 这个数字是怎么来的？能不能带我走一遍你的计算过程？500 这个数是最差情况、最佳估计、还是确认数字？"
- "你查过所有相关库存地点了吗？（成品仓/半成品仓/线边仓/在途/客户 VMI 仓/客户未上线库存）哪一个你可能漏了？"

**Layer 2 — 分层分析**
- "把缺陷率按批次/产线/班次/模穴分开来看，分布是均匀的还是集中的？如果某个子集缺陷率是 0%，同时另一个子集是 30%，这说明什么？"
- "如果有按时间分布的数据，读 `docs/patterns/temporal-patterns.md` 做时间模式识别。突然跃升？逐渐恶化？间歇性？周期性？模式本身缩小 80% 搜索范围。"

**Layer 3 — 时间模式分析**
- "除了已经确认不良的，有没有一些产品是'可疑的'或'边界上的'？这些在计数里怎么处理？"

**Layer 3 — 商业影响**
- "如果你用 PPM 和成本两个维度给这个问题打分，它在你过去一年处理的客户投诉里排第几位？这个排名准确吗？"
- "如果客户因为这个问题切换了供应商，年化损失是多少？这个概率你估计有多大？"

## S3 | 反事实挑战

1. "假设你现在的数量估计偏低了 2 倍（实际情况比你以为的严重），最可能漏算的渠道或位置是哪里？为什么？"
2. "如果客户有一天来审计，说你的缺陷率数据和他们的来料检验数据对不上，你觉得差异最可能出在哪？（你们的全检有效性？客户的抽样方法？产品在运输后发生了变化？）"

## S4 | 答案评分

评分维度：**数据准确性**、**覆盖完整性**、**商业关联性**

| 等级 | 标准 |
|------|------|
| Insufficient | 无具体数字（"不少""挺多"）；只统计了客户退回数量；无缺陷率计算；无成本估算 |
| Basic | 数量有统计来源（至少客户退回+内部库存）；缺陷率已计算；主要成本项已估算；风险敞口已标识 |
| Excellent | 所有库存/在途/WIP/客户仓均已盘查；按批次/产线等维度分层分析缺陷分布；直接成本+间接成本+风险敞口完整；数据来源可追溯（系统记录/盘点记录） |

## 纠偏原则 / 禁止项

| 禁止 | 替代方向 |
|------|---------|
| 只统计客户退回的数量 | 盘查所有风险窗口：成品仓/WIP/在途/客户未用库存 |
| 用"大概""估计"代替数字 | 即使是不确定的数字，标注范围（100-200）也比形容词好 |
| 缺陷率只看总体不看分层 | 按批次/产线/班次分层，寻找集中度模式 |
| 忽略成本 | 成本数据让管理层和客户理解严重度的共同语言 |
| 高估围堵覆盖率 | 诚实列出还没查到的库存位置，标注为风险敞口 |
