# d7-q1 — 同类检查（Yokoten）

## 行业背景

Yokoten 源自丰田生产系统，字面意思是"横向展开"，核心原则是：在一个点上发现的问题，必须主动检查所有相似点是否存在同样风险。IATF 16949:2016 第 10.2.3 条（问题解决）明确要求组织定义过程以防再发，包括将纠正措施应用于类似过程和产品。VDA 8D 标准将 D7 定义为"预防复发"，同类检查是其中第一道防线——如果只在当前产品上改了，隔壁产线明天出一模一样的问题，这个 8D 就白做了。

同类检查的范围不是"所有产品"，而是有明确定义的相似维度：相同结构设计、相同工艺参数、相同物料/来料、相同供应商、相同设备型号、相同检测方法。检查必须产出清单，而不是一句"已检查完毕"。

## S1 | 发散探索 — 多角度触发器

从以下角度逐一提问，确保同类检查不遗漏：

1. **结构维度**：哪些产品/零件与本次问题件共用同一个设计结构？同一个图纸族？同一个装配关系？列产品编号或型号，不要只说"同类产品"。

2. **工艺维度**：哪些产线/工序使用了与本次问题点相同的工艺参数（温度、压力、时间、速度）？相同的工装/夹具/模具？

3. **物料维度**：同一批号/同一供应商的物料还流向了哪些产品、哪些工单？来料检验记录中是否有同批次的异常信号？

4. **设备维度**：同型号设备在其他产线上是否运行着相同的程序/参数？设备的 PM 记录中是否有类似趋势？

5. **供应商维度**：同一供应商是否向其他产品线供货？该供应商的 PPM/投诉趋势是否在恶化？是否需要在供应商处做联合检查？

## S2 | 第一性原理收敛

追问到系统和组织层面：

- 为什么这个风险点之前没有被识别？是 FMEA 里没有覆盖这个失效模式，还是覆盖了但没有有效的探测控制？
- 变更管理流程中，为什么这个同类风险没有被自动触发？变更评估清单是否缺少"同类产品影响"这一项？
- 如果同类检查发现了风险，说明什么：是设计规范有系统性问题，还是工艺标准在不同产线上执行不一致？
- 如果同类检查没有发现风险，边界在哪里：是什么设计/工艺差异让其他产品天然免疫？这个差异能否反向加固当前产品？

## S3 | 反事实挑战

- 如果我们只改了当前产品/产线，不改同类项，6 个月后客户投诉的概率有多大？拿什么数据支撑这个判断？
- 如果同类产品/产线已经跑了几万件没出问题，是真的没问题，还是没被检测到？检测手段有差异吗？
- 假设供应商告诉我们"其他客户都没投诉"，我们怎么独立验证？有没有办法绕过供应商的嘴，直接看数据？
- 如果资源只够检查 3 个同类项中的 1 个，你根据什么逻辑选优先级？这个逻辑经得起质疑吗？

## S4 | 答案评分

| 维度 | Insufficient | Basic | Excellent |
|------|-------------|-------|-----------|
| **范围具体性** | "已检查所有同类产品"——没有清单，没有产品名 | 列出了检查的产品/工序/供应商清单 | 清单 + 每项的检查方法 + 检查结果 + 证据 |
| **逻辑完整性** | 只查了最容易查的一项 | 覆盖了结构/工艺/物料三个以上维度 | 覆盖全部五个维度，每个维度有明确的纳入/排除逻辑 |
| **行动闭环** | 发现了风险但没有处置动作 | 对确认的风险启动了围堵 | 围堵 + 责任人 + deadline + 状态跟踪 |
| **反向验证** | 没有考虑"为什么别的没出问题" | 分析了差异但停留在表面 | 识别出保护因子并可反向加固 |

不达 Basic 不能进入 S5。至少 3 个维度达 Basic 以上。

## 纠偏原则 / 禁止项

- **禁止**："已检查所有同类产品/工序"——没有清单等于没检查。
- **禁止**：只写检查范围不写检查方法。光说"看了"但在哪看、看什么数据、怎么看，都没交代。
- **禁止**：发现同类风险后只记录不上报、不围堵。Yokoten 的核心是从发现到行动。
- **禁止**：以"其他产品没问题"为由跳过检查。没检查过就不能说没问题。
- 同类检查是预防的最后一道门。如果这里漏了，D5 的措施只在一条产线上有效，整个 8D 的投资回报打三折。
