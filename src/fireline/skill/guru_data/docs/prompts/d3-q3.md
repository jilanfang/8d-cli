# d3-q3 — 库存产品围堵

## 行业背景

IATF 16949 第 8.7.1 条要求在围堵阶段冻结所有受影响批次的库存，无论成品还是半成品。AIAG CQI-20 强调：库存围堵的难点不是"围起来"，而是区分"哪些库存涉及、哪些不涉及"——这个判断依赖于批次的精确追溯。ASQ 指出，很多组织的批次追溯在成品端完整，但到半成品和中间品就断了——因为不同工序使用了不同的批次命名规则，追溯链条在中间断裂。Ford 的 Q1 审核要求：成品和半成品的围堵措施必须分别描述，因为处置方式和风险不同。

## S1 | 发散探索 — 多角度触发器

1. **你的成品库存和半成品库存在哪里？** 几个仓库？哪个仓库有受影响批次？你是按批次号查的，还是按生产日期反推的？
2. **批次追溯链条在成品和半成品之间是连续的吗？** 如果你能追溯到成品批号，能不能一路追溯到对应的半成品批号？中间有没有换批/混批的工序？
3. **你怎么知道你锁定的就是100%的受影响库存？** 有没有盘点过实物？系统数量和实物数量一致吗？有没有搬到临时库位的货没在系统里更新？
4. **半成品库存的处理方式不同于成品吗？** 半成品能不能直接筛选，还是必须先做完后续工序才能判定？如果必须做完才能判，你允许继续投料做下去吗——还是先冻结整个批号？
5. **不涉及的库存你是怎么排除的？** 相邻批次、同一天生产的其他产品、同一产线不同班次——这些你排除了吗，依据是什么？

## S2 | 第一性原理收敛

追问链（对涉及的库存窗口逐条追问）：

- 你说的"成品库存已经冻结"，冻结是怎么执行的？是系统锁库还是物理贴封条？谁有权限解冻？解冻需要谁审批？
- 冻结的产品现在放在哪里？是专门的隔离区，还是只是在原库位上贴了个标签？如果是贴标签，会不会有人误用——标签掉了怎么办？
- 半成品如果必须继续做完才能判定，你设了什么控制条件？每一件继续投料的半成品都有追踪号吗？做完了判定不合格谁能拦下来？
- 你的隔离品区域有没有做到"物理隔离"？一个围栏、一个锁起来的区域、还是一个独立的仓库？如果只是在地上画了条线，线以内的货被叉车叉走了怎么办？
- 冻结的库存最终怎么处置？有没有一个明确的处置决策日期？谁有权力决定这批货是报废、返工、还是让步接收？

## S3 | 反事实挑战

"如果你的库存冻结失败了——系统锁了库但仓库没执行、或者隔离区标签掉了混回了正常品——这批可疑品混在下批出货里到了客户手里。你的下一个防线在哪里？是出货前最后一道检验——但那道检验是在查什么，能查出这个问题吗？还是你的质量追溯系统能在客户投诉时发现'这批货其实是冻结品'？"

## S4 | 答案评分

| 维度 | 不通过（0分） | 合格（1分） | 深入（2分） |
| :------ | :--------- | :------- | :------- |
| **批次追溯完整性** | "成品库存已锁定"，未提半成品 | 成品+半成品分别说明，追溯逻辑清晰 | 成品→半成品→原料全链路追溯+断点识别+系统/实物校验 |
| **隔离措施可靠性** | "已冻结"没有物理隔离细节 | 有隔离区域+标识+权限控制 | 有物理隔离+防误用机制+解冻审批流程+定期巡查 |
| **处置决策明确性** | "待处理"没有时限和决策人 | 有时限、有决策人、有处置选项 | 有时限+决策人+处置选项分析+成本估算+风险矩阵 |

## 纠偏原则 / 禁止项

- **禁止只围成品不围半成品。** 半成品也是库存。如果问题工序在半成品之后，半成品就是直接受影响品。即使半成品需要后续工序才能判定，也必须说明后续工序的控制方式和责任人。
- **禁止"系统锁库"当作唯一的冻结措施。** 物理隔离是必要的——系统锁库防止发货，但防止不了仓库人员因为紧急订单从货架取走。必须说明物理上产品放在了哪里、怎么防止误用。
- **禁止缺少处置决策时间线。** 冻结不是终点。冻结的产品最终要销毁、返工还是让步，这个决策如果没有截止日期，隔离区会越堆越多，最后变成历史遗留问题。
- **禁止用"库存已处理"跳过处置细节。** "已处理"是三个没有信息的字。必须写清楚：怎么处理的、谁处理的、什么时候处理的、处理了多少件、处理结果是什么。
