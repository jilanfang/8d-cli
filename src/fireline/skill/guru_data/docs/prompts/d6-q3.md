# d6-q3 — 验证接受标准

## 行业背景

IATF 16949 要求验证标准必须在执行前定义（pre-defined acceptance criteria），这是确保验证不成为走过场的关键机制。VDA 第 8 卷明确：如果验证标准在执行后才制定，或者执行过程中调整标准以匹配结果，那就是“编结论”而不是“做验证”。

AIAG CQI-20 指出 D6 验证标准有三种主要形式：
1. 基于能力：Cpk ≥ 1.33 / Ppk ≥ 1.67
2. 基于计数：连续 N 个批次 0 缺陷 / 不良率从 X ppm 降至 ≤ Y ppm
3. 基于时间：客户现场 N 天/月无投诉复发

标准必须量化、可观测、无歧义。不允许“质量明显提升”“客户满意”“情况好转”这类主观判断。

标准的另一个关键维度是对称性：不仅要定义“通过”的标准，也要定义“不通过”的标准和“灰色地带”怎么处理。不能只有 pass 没有 fail。

## S1 | 发散探索 — 多角度触发器

1. 接受标准的数值是怎么定出来的？基于客户要求、行业基准、历史能力还是统计计算？
2. 如果验证结果刚好踩在线上（比如 Cpk 正好 1.33），过不过？有没有缓冲区？
3. 你定义了“不通过”的标准，但你定义了什么情况下算“不确定/需要更多数据”吗？
4. 每个验证方法都有对应的接受标准吗？还是只有一个笼统的全局标准？
5. 如果短期验证通过但中期趋势显示改善不稳定（比如 Cpk 1.33 但波动在变大），你接受还是不接受？

## S2 | 第一性原理收敛

从统计学和工程判据角度追问接受标准的严谨性：

- **统计置信与工程裕度**：接受标准不能只靠一个点估计。例如“Cpk ≥ 1.33”需要同时要求 Cpk 的置信下限也满足（通常 Cpk 置信下限 ≥ 1.0，基于 95% 置信区间）。单个数据点应要求置信区间而非点估计。
- **多指标一致性**：如果措施对应的验证指标有多个（比如同时要求 Cpk 和连续 0 缺陷），这些指标之间是否一致？有没有可能出现一个指标通过另一个不通过的情况？那时怎么裁定？
- **可操作性定义**：每个接受标准必须有清晰的操作定义——谁来判断、用什么数据、什么时间判断、数据口径是什么。不能事后争论“这个算不算不良”。
- **客户视角校准**：技术指标（如 Cpk ≥ 1.33）和客户使用中的体验（如 0 投诉）之间的映射是否经过验证？如果 Cpk 满足了但客户还在投诉，说明指标选择有问题。

灰色地带处理规则：如果验证结果落入“不确定”区间（比如 Cpk 1.20-1.33 之间），默认判定为不通过并延长验证周期，而不是默认通过。

## S3 | 反事实挑战

- 如果验证结果是连续 3 批 0 不良（满足标准），但第 4 批出现 1 个不良，是验证不够长、标准不够高还是措施不够稳？
- 如果你自己是客户的 SQE（供应商质量工程师），看到这个接受标准，你会觉得靠谱吗？还是会觉得供应商在给自己放水？
- 最严格最怀疑这个 8D 的人会怎么质疑你的接受标准？他可能会说什么？你怎么回应？

## S4 | 答案评分

| 维度 | 1分 | 3分 | 5分 |
|------|-----|-----|-----|
| 可量化性 | 标准含主观判断词 | 有量化标准但边界模糊 | 标准清晰量化，包含上下界 |
| 统计严谨性 | 只有点估计，无置信区间 | 有置信区间考虑但不完整 | 点估计+置信区间判断规则完整 |
| 对称完整性 | 只有通过标准 | 有通过和失败标准 | 通过/失败/不确定三级标准且处置规则明确 |

三项均 ≥ 3 分才进入下一步。

## 纠偏原则 / 禁止项

- **强制拒绝的接受标准表述**：
  - “没有不良” → 必须写“连续 N（具体数字）批次/件 0 不良”
  - “质量改善” → 必须写“XX 指标从 Y 改善到 Z（量化值）”
  - “客户满意” → 必须写“客户书面确认无同类投诉且连续 N 天无新增投诉”
  - “没问题了”“看起来正常”“解决了”
- 接受标准在 D5 阶段设定，D6 阶段只能遵守，不能调整。如果数据证明标准不合理，记录原因后走变更流程，不能直接改标准来匹配数据。
- 必须有 fail criteria 和处置规则。不能只有 pass 没有 fail。
