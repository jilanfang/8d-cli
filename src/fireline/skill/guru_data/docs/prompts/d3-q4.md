# d3-q4 — WIP 围堵

## 行业背景

IATF 16949 第 8.7.1 条要求对生产线上的在制品（WIP）立即围堵——停产、加严检验或 100% 筛选。AIAG CQI-20 指出，WIP 围堵的最大风险是"产线继续跑"：在问题确认和围堵指令到达产线之间有一个时间差，这个时间差内生产的产品也会进入受影响池。ASQ 建议：WIP 围堵指令必须是"看得见的"——工位上的物理标识、系统里的生产暂停、以及班前会的口头传达三者缺一不可。Ford 要求 WIP 围堵必须精确到工序级别：哪几个工序的 WIP 受影响、每个工序的处置方式是什么。

## S1 | 发散探索 — 多角度触发器

1. **你的产线现在还在生产吗？** 你是在问题确认后立即停机，还是继续生产但加了检验？如果继续生产，凭什么相信加了检验就能拦住？
2. **从问题确认到产线停线/加严检验，中间隔了多久？** 这个时间差里生产出来的产品怎么算——算在受影响池内，还是单独判定？你现在能精确追溯到这个时间窗口内的产品吗？
3. **受影响的是哪几个工序？** 只是问题发生的那个工序，还是也包括上游已经做完了等待进入问题工序的产品，以及下游已经流出去的产品？你在每个工序区分了"已生产完成待流出""正在加工""已投料未加工"三种状态的 WIP 吗？
4. **如果不停产，你在关键工序加了什么检验？** 检验方法能检出这个问题吗——检测能力有没有验证过？抽检还是全检？抽检的 AQL 是多少——这个 AQL 能保护客户吗？
5. **产线有多少个工位、多少件 WIP？** 你是在系统里查了产线报工记录，还是派人在产线上逐件清点的？两个数字一致吗？

## S2 | 第一性原理收敛

追问链（对涉及的 WIP 窗口逐条追问）：

- 你说的"已经在问题工序加了100%检验"，这个检验方法做过盲测吗？拿一批已知有问题的产品和一批已知没问题的产品混在一起，你的检验员/设备能不能全部检出来？
- 如果加严检验也拦不住（比如问题不易检出、需要破坏性测试），你的 Plan B 是停产还是扩大筛选范围到上游工序？
- 上游工序已经生产完成但在制品的产品，你判定它"不受影响"的依据是什么？是因为问题只在下游工序发生、上游产品还没经过问题工序吗？如果问题根因排查还没完成，你怎么确定上游不需要围堵？
- 围堵期间的产线生产计划怎么调整？如果停产三天，客户的交付怎么办——你通知客户交期延迟了吗？你评估过停线和继续出问题哪个对客户的影响更大？
- 重新开线需要谁的批准？复线条件是什么——围堵措施到位？根因找到？纠正措施完成？还是有临时措施就可以先恢复生产？

## S3 | 反事实挑战

"如果你的 WIP 围堵失败了——你在问题工序加的检验漏检了、或者围堵指令下达前那批产品没有被追回来——这批产品继续往下流，变成了成品库存，混进了客户的下一批出货里。到那个时候，你还能在哪个环节拦住它？是成品出厂检验——但那道检验是为这个失效模式设计的吗？还是客户的来料检验——他们会查这个特性吗？"

## S4 | 答案评分

| 维度 | 不通过（0分） | 合格（1分） | 深入（2分） |
| :------ | :--------- | :------- | :------- |
| **工序级精准度** | "产线 WIP 已围堵"未区分工序 | 按工序逐一说明，区分了三种 WIP 状态 | 按工序+工位+状态三维说明+每件追溯号+时间窗口产品定位 |
| **检验能力验证** | "加了检验"但未说明方法 | 说明了检验方法+抽检/全检+判定标准 | 有检验方法+盲测验证+检出率数据+漏检风险评估 |
| **产线管控完整性** | 只有围堵，没有复线条件 | 有暂停/复线决策标准 | 有暂停+复线审批+围堵期间生产计划调整+客户交期影响评估 |

## 纠偏原则 / 禁止项

- **禁止"产线已停机"但没有说明停在哪几个工位。** 整条线停还是部分工位停？上游停了但下游还在跑吗？每个工位的状态必须单独说明。
- **禁止"加了检验"但不说明检验的有效性。** 检验不是装饰。加检验不意味着能检出问题。必须追问：这个检验方法对这个失效模式的检出率是多少？有没有数据支持？
- **禁止忽略"时间窗口产品"。** 从问题确认到围堵指令实际到达产线，中间一定有时间差。这段时间生产的产品是最危险的一批——它们没有经过围堵检验，但也没有被标记。必须追踪这批产品的去向。
- **禁止把"操作员自己检查"当作围堵检验措施。** 操作员自检不是围堵。围堵检验必须是独立的、有记录的、和正常生产检验区分开的。自检的标准、频次、可靠性都不可靠。
