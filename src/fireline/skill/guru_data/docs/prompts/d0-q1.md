# d0-q1 — 现象描述

## 行业背景

D0 的第一步是把"客户说了什么"转换成"产品实际发生了什么"。Ford 8D 手册和 AIAG CQI-20 都强调：症状描述必须基于可观察事实，而非推测性根因。把根因写进现象描述是 8D 最常见的早期错误 —— 一旦"因为焊接不良导致无输出"被当成现象写进 D0，D4 根因分析的搜索空间就被窄化为"为什么焊接不良"，真正的根因（比如设计应力集中、来料批次变异、测试覆盖漏洞）可能永远不会被追问。IATF 16949 审核中，现象与根因混淆是高频不符合项。D0 的现象准确度直接决定 D2 Is/Is-Not 矩阵的质量和 D4 根因搜索方向。

## S1 | 发散探索 — 多角度触发器

Agent 从以下问题中选 2-3 个向用户提问。

1. **感官还原**："如果你当时站在产品旁边，你的眼睛看到了什么？耳朵听到了什么？手摸到了什么？鼻子闻到了什么？不要说'失效了'，说具体现象。"
2. **正常对照**："同样的产品，正常的时候是什么表现？异常和正常的差异在哪个具体时刻、哪个具体部位出现？"
3. **可复现性**："这个现象是一直存在还是间歇性的？如果再做一次同样的操作，现象会复现吗？在什么条件下复现，什么条件下不出现？"
4. **环境上下文**："现象发生时，周围环境有什么值得注意的？温度？湿度？振动？刚换班？刚换料？刚调过参数？"
5. **时间序列**："从正常运行到出现异常，中间发生了什么事件？有没有一个明确的转折点？还是逐渐变差的？"

## S2 | 第一性原理收敛

追问链 —— 从表面描述逐层深入：

**Layer 1 — 模糊到具体**
- "你说'产品不工作'—— 是完全不工作还是部分功能不工作？哪个功能？用什么方法确认的？"
- "你说'有异响'—— 是什么样的声音？摩擦声、撞击声、电流声？在哪个位置？能不能录音？"
- "你说'外观有问题'—— 具体是划痕、变形、变色、裂纹、还是异物？尺寸多少？位置在哪？"

**Layer 2 — 测量化**
- "这个现象有没有被测量或量化？比如：电压从 5V 掉到多少？噪音多少分贝？尺寸偏差多少毫米？"
- "如果没有测量数据，有没有照片或视频？能不能现在就让人去拍？"
- "你说的'温度高'—— 多少度？用什么测的？测的哪个点？正常范围是多少？"

**Layer 3 — 边界条件**
- "这个现象在什么条件下一定会出现？在什么条件下一定不会出现？"
- "同一批产品里，是所有产品都这样，还是只有部分？是哪一个部分？"

## S3 | 反事实挑战

1. "假设你描述的'现象'其实不是产品本身的问题，而是测试设备、测量方法、或观察条件造成的假象 —— 你用什么方法能在 10 分钟内排除这种可能？"
2. "如果客户描述的'现象'实际上是两个不同的问题叠加在一起的表现（比如外观划痕 + 功能失效是两个独立事件），你怎么证明它们是同一个问题还是两个问题？"

## S4 | 答案评分

评分维度：**可观察性**、**具体性**、**可复现性**

| 等级 | 标准 |
|------|------|
| Insufficient | 现象描述混入了根因推测（如"因为焊接不良"）；使用模糊词汇（"坏了""有问题""不正常"）；无任何测量数据或影像记录 |
| Basic | 现象用感官语言描述（看到/听到/摸到）；与正常状态有对比；说明了发生条件；有照片/视频或承认缺失 |
| Excellent | 现象可量化（数值/单位）；边界条件明确（什么条件下出现/不出现）；有时序信息（从正常到异常的转折点）；排除了测量系统干扰 |

## 纠偏原则 / 禁止项

| 禁止 | 替代方向 |
|------|---------|
| 把根因写到现象里（"因为XX导致YY"） | 只描述可观察的 YY。XX 留给 D4 |
| "产品坏了" / "功能失效" / "质量问题" 这些空泛标签 | 追问具体功能、具体表现、具体部位 |
| 接受没有证据的现象描述 | 至少要求照片或视频。实在没有，记录"无证据，仅口头描述" |
| 把客户的总结当成现象 | 追问客户"具体看到了什么"，引用客户原话 |
| 用内部工程术语改写客户描述 | 保留客户原话 + 附工程翻译，两者并存 |
| 跳过环境/时序信息 | 现象从来不是孤立发生的，必须记录上下文 |
