# d7-q2 — 系统文件更新

## 行业背景

IATF 16949:2016 第 7.5.3.2.1 条要求：当过程发生变更时，相关的过程文件（PFMEA、控制计划、作业指导书）必须同步更新。AIAG/VDA FMEA 手册（2019 第一版）明确规定了 PFMEA 和控制计划的联动关系——PFMEA 中的 RPN/AP 评估结果直接影响控制计划的探测和反应策略，两者之间的信息流必须是双向且一致的。

在 8D 实践中，D7 文件更新最常见的失败模式是"局部修复"：只改了一份 SOP，但 PFMEA 还是旧的发生频度、控制计划还是旧的探测方法、检验标准还是旧的允收准则。三个月后工艺又变回去了，因为只有一份文件改了，其他文件没有形成合力。

系统文件更新的核心原则：**所有受影响的文件必须在同一个时间窗口内完成更新，并且内容一致**。不能 FMEA 说"已增加防错装置"，控制计划里却没有这项控制。

## S1 | 发散探索 — 多角度触发器

从以下角度逐一确认需要更新的文件：

1. **PFMEA / DFMEA 维度**：本次失效模式在 FMEA 中是否已覆盖？如果已覆盖，发生频度（O）和探测度（D）的当前评分是否被实际发生的事实所推翻？如果未覆盖，需要新增哪一行？现行控制（Prevention Control 和 Detection Control）是否需要修正？

2. **控制计划维度**：控制计划中对应工序的控制方法、样本量、频次是否需要调整？是否需要增加新的控制点（例如从首末件检查升级为每件自动检测）？

3. **作业指导书（SOP）维度**：操作步骤、关键参数、点检要求是否需要更新？操作员是否已经按新方法执行？旧版 SOP 是否已经从现场回收？

4. **检验标准/检验指导书维度**：允收标准是否需要收严？检验频次是否需要增加？是否需要增加新的检验项目或检验工具？

5. **设备维护/工装管理维度**：设备参数是否需要锁定/防错？PM 计划是否需要增加检查项或缩短周期？工装/模具是否需要修改并重新验证？

## S2 | 第一性原理收敛

追问到系统层面：

- 为什么这次失效在 FMEA 中没有被有效预防？是 FMEA 当时做浅了（打分过于乐观），还是 FMEA 做了但控制计划没有承接？
- 文件更新的审核机制可靠吗：谁负责检查"所有相关文件都已同步"？有没有一个 checklist 确保 FMEA、控制计划、SOP、检验标准之间的交叉引用一致？
- 如果这次改了三份文件，下次类似变更时，工程师能从哪些文件中找到完整逻辑？还是改完之后知识只存在工程师脑子里？
- 文件更新后的培训如何验证？不是签个培训记录表就算完了——操作员能不能在新旧两版之间准确识别差异点？

## S3 | 反事实挑战

- 如果只改 SOP 不改 PFMEA，下一次 PFMEA 评审时（可能是两年后），评审团队还能不能看到这次失效的教训？如果看不到，意味着这个知识存活期只有两年。
- 如果只改控制计划不改检验标准，IQC/IPQC/OQC 还在按旧标准放行，新控制计划只是墙上的纸——怎么保证落地？
- 如果这次改的文件明年被 ISO/TS 审核抽到，审核员能在三份文件（FMEA、控制计划、SOP）之间找到一致的信息吗？
- 假设责任人离职了，新来的人能不能看懂这些文件更新的逻辑和关联？还是只能看到单份文件的修订记录？

## S4 | 答案评分

| 维度 | Insufficient | Basic | Excellent |
|------|-------------|-------|-----------|
| **文件覆盖完整性** | 只提了 1 份需要更新的文件 | 列出了 FMEA + 控制计划 + SOP/检验标准 | 涵盖全部受影响文件 + 说明了文件间的交叉引用关系 |
| **变更具体性** | "更新 FMEA"——没有说改哪一行、改什么 | 说明了具体变更内容（改哪一项、从什么改到什么） | 变更内容 + 变更理由 + 关联文件的一致性说明 |
| **同步性保证** | 各文件各自更新，没有同步机制 | 明确了同步窗口和审核人 | 同步机制 + 交叉引用验证 + 文件版本关联（如"随 PFMEA Rev.12 同步更新"） |
| **落地闭环** | 只有文件更新计划，没有培训/回收旧版 | 包含了旧版回收和培训计划 | 培训 + 验证（现场确认操作员理解差异点）+ 旧版回收确认 |

不达 Basic 不能进入 S5。PFMEA 和控制计划必须同时出现在更新清单中，否则直接 Insufficient。

## 纠偏原则 / 禁止项

- **禁止**：只更新一份文件。FMEA、控制计划、SOP、检验标准如有任一需要更新，所有相关文件必须同步。
- **禁止**："更新 FMEA"——不说什么工序、什么失效模式、改什么，等于没写。
- **禁止**：用"加强"、"完善"、"优化"这类词描述变更。必须写清楚从什么改到什么。
- **禁止**：文件更新了但是旧版还在现场。旧版文件回收是更新闭环的一部分，不能省略。
- PFMEA 和控制计划如不同步，整个 D7 评为 Insufficient，必须在 gate 中 block。
