# d2-q3 — 问题边界（Is / Is-Not）

## 行业背景

Is/Is-Not 矩阵是 D2 最核心的分析工具，也是 D4 根因缩小搜索范围的基础。Kepner-Tregoe 方法论（ASQ 和 Ford 8D 的底层框架）要求从 What/Where/When/Who/How Many 五个维度同时建立 IS（问题是什么）和 IS-NOT（问题不是什么，但在逻辑上可能是）的对比。这个工具的精髓在于：列出来的 IS-NOT 和 IS 来自同一个逻辑类别。不是随便写"不是别的问题"，而是"同一批次的其他产品不是"、"同一产线的上一班次不是"。IATF 16949 审核中，Is/Is-Not 矩阵有 IS 无 IS-NOT 或 IS-NOT 不具体，都是常见不符合项。没有 IS-NOT 的 Is/Is-Not 矩阵不是矩阵，只是一个问题清单。

## S1 | 发散探索 — 多角度触发器

Agent 从以下问题中选 2-3 个向用户提问。

1. **What 维度**："问题的具体失效是什么？什么功能/特性出了问题？反过来问：什么功能/特性没有出问题，虽然看起来可能相关？"
2. **Where 维度**："问题发生在产品的哪个位置/哪个部位？发生在哪个客户/哪个市场？发生在哪条产线/哪个工序？反过来问：哪个位置/部位没有发生？哪个客户/市场没有投诉？哪条产线没有出现？"
3. **When 维度**："问题是从什么时候开始出现的？精确到班次或日期。在产品的生命周期哪个阶段出现的（来料/制造/测试/运输/客户使用）？反过来问：什么时间之前没有？哪个阶段没有出现？"
4. **Who/How Many 维度**："多少产品受影响？绝对数量和比例各是多少？哪些客户/批次/产线/工位受到影响？反过来问：哪些批次确认不受影响？（需要明确列出不受影响的批次号）"
5. **边界矛盾探测**："有没有你觉得'应该是'但实际'不是'的情况？比如你预计某个批次也会有问题但实际上没有？这种反直觉的 IS-NOT 是最有价值的线索。"

## S2 | 第一性原理收敛

追问链 —— 从表面信息逐层深入：

**Layer 1 — IS-NOT 质量检查**
- "你写的 IS-NOT 和 IS 是不是同一个逻辑类别？比如 IS 写了'批次 2403'，IS-NOT 写'不是设计问题'—— 这不对。应该是'批次 2401、2402、2404 不是'。"
- "每一个 IS-NOT 你确认过了吗？还是你觉得'应该不是'但没有实际检查过？"

**Layer 2 — 对比分析**
- "IS 和 IS-NOT 之间有什么关键差异？如果一个工位有问题、相邻工位没问题，这两个工位有什么不同？（设备/人员/物料批次/作业方法）"
- "如果可以找到一个'几乎相同但没有问题'的对照物（同一产品、同一产线、同一天、但不同班次/不同模穴/不同材料卷号），差异是什么？"

**Layer 3 — 边界条件**
- "问题发生的最早时间点和最晚时间点是什么？这两个时间点之间发生了什么变化？（换料/换班/调参/设备维护/软件更新）"
- "有没有一个明确的截止边界 —— 某个时间点之后的产品全部有问题，之前的全部没问题？如果有，那天/那个班次发生了什么？"

## S3 | 反事实挑战

1. "你现有的 IS-NOT 列表里，有多少是'有证据确认不是'，有多少是'感觉应该不是但没查过'？如果把'感觉系 IS-NOT'标出来，去验证其中三个，你的 IS/IS-NOT 矩阵会不会变？"
2. "如果一个外部专家看了你的 IS 列表，他能不能从 IS-NOT 列表里挑出一个你认为'不是'但实际上是'是'的项目？哪个 IS-NOT 最脆弱？"

## S4 | 答案评分

评分维度：**IS-NOT 质量**、**对比清晰度**、**边界精确度**

| 等级 | 标准 |
|------|------|
| Insufficient | 只有 IS 没有 IS-NOT；IS-NOT 模糊（"不是其他问题"）；IS/IS-NOT 不在同一逻辑类别；无时间/批次边界 |
| Basic | 五个维度均有 IS 和 IS-NOT；IS-NOT 与 IS 在同一逻辑类别；时间/批次有具体范围；关键 IS-NOT 有验证 |
| Excellent | 每个 IS-NOT 有验证来源（查过/测过/确认过）；找出了 IS 与 IS-NOT 之间的关键差异变量；时间边界精确到班次；矛盾线索（反直觉的 IS-NOT）已标记并用于缩小 D4 搜索范围 |

## 纠偏原则 / 禁止项

| 禁止 | 替代方向 |
|------|---------|
| IS-NOT 写成"不是其他问题""不是设计问题" | IS-NOT 必须和 IS 在同一逻辑类别。IS 是"批次 A"，IS-NOT 必须是"批次 B、C、D" |
| IS-NOT 凭感觉写，不验证 | 每个 IS-NOT 标注验证状态：已确认 / 推测未验 / 待查 |
| 只用一个维度（what） | 必须五维度全部覆盖：What/Where/When/Who/How Many |
| 边界粒度太粗（"最近几个月"） | 精确到班次或日期。越精确，D4 搜索范围越小 |
| 忽略"反直觉的 IS-NOT" | 预计应该有问题但实际没有的线索，往往指向根因的关键变量 |
