# D2 时间模式检测

> Agent：在 D2-q4 收集到按时间分布的不良数据后，加载本文件帮助用户识别时间模式。模式本身能缩小 80% 的 D4 根因搜索范围。

## 为什么时间模式重要

不良率的**时间形状**本身就是最强的根因线索。同样的不良数量（100 台），如果集中在某一天突然出现 vs 三个月内缓慢恶化 vs 每隔两周周期性出现，指向的根因完全不同。不先看模式就直接进 D4 6M 分析，等于放弃了一个免费的高价值信息源。

## 六种模式

### 1. 突然跃升（Sudden Step Change）

**特征**：某个明确时间点之前不良率趋近于零，之后突然跳到高位并保持。

**数据形状**：
```
不良率
  |          ████████████████
  |          ████████████████
  |__________████████████████
  └────────────────────────── 时间
        ↑ 突变点
```

**指向根因**：在那个突变时间点发生了某个**变更事件**。可能是换料批次、设备维修/换件、参数调整、软件更新、新操作员上岗、模具更换。这不是系统性退化，是一个事件触发的。

**Agent 追问**：
- "突变点那天/那个班次发生了什么变更？人、机、料、法——列出所有变更，哪怕是'看起来无关的'。"
- "突变前后的产品，除了不良率不同，还有什么可测量的差异？"
- "突变点之后有没有一次维修/调整/回调？如果有，回调后不良率变了吗？"

### 2. 逐渐恶化（Gradual Drift / Degradation）

**特征**：不良率随时间持续上升，斜率可能稳定也可能加速。

**数据形状**：
```
不良率
  |                  ████
  |              ████
  |          ████
  |      ████
  |__████
  └────────────────────────── 时间
```

**指向根因**：系统在**退化**。工具磨损、设备漂移、材料性能衰减、环境季节性变化。这不是"什么事突然变了"，而是"某件事一直在变但没人监控"。

**Agent 追问**：
- "什么特性会随时间/产量/使用次数退化？刀具？模具？过滤器？传感器？"
- "退化速率和什么变量相关——产量、时间、还是环境条件？"
- "为什么不定期维护/更换没有阻止这个退化？维护周期设对了吗？"

### 3. 间歇性（Intermittent / Sporadic）

**特征**：不良偶尔出现，没有明显的时间规律。有时连续几批为零，有时突然出现几个。

**数据形状**：
```
不良率
  |  █  ██    █    ███  █
  |__█__██__█_█__█_███__█__ 时间
```

**指向根因**：**接触不良、松动、环境波动、边界条件触发**。问题不是一直存在的，只在特定条件组合下出现。这是最难找根因的模式。

**Agent 追问**：
- "不良发生在什么特定条件下？哪个班次？哪个机台？哪种来料批次？什么温湿度？"
- "能画出'不良发生时的条件交集图'吗——同时具备哪些条件时不良一定出现？"
- "有没有可能是不良的测量系统本身不稳定（不是产品问题，是测量问题）？"

### 4. 浴缸曲线（Bathtub Curve）

**特征**：早期不良率高 → 中间稳定低 → 晚期再次升高。

**数据形状**：
```
不良率
  | ███                   ████
  | ███       ████████    ████
  | ███___________________████
  └────────────────────────── 时间
    早期        中期        晚期
```

**指向根因**：**两个不同根因叠加**。早期的根因和晚期的根因通常不是同一个问题。早期 → 新模具、新材料、新操作员上手的调试期问题。晚期 → 磨损、老化、维护周期到期。

**Agent 追问**：
- "你确定早期的不良和现在的不良是同一个失效模式吗？有没有可能根因不同？"
- "早期不良是怎么处理的——当时做了调整还是等它自己消失了？"

### 5. 周期性（Cyclical / Seasonal）

**特征**：不良率按固定间隔波动——每班、每天、每周、每月、每季度。

**数据形状**：
```
不良率
  |  █     █     █     █
  | ███   ███   ███   ███
  |_█████_█████_█████_█████ 时间
     周一周二..
```

**指向根因**：与**时间节奏相关**。换班（夜班 vs 白班）、周保养后的重启、月结冲刺时的赶工、季节性温湿度变化、周期性来料切换。

**Agent 追问**：
- "周期是多少？天/周/月？这个周期和你工厂里的哪个节奏重合？换班？保养？投料？报表截止？"
- "不良在周期内哪个时点最高？刚换班后？快下班前？刚做完保养后第一次生产？"

### 6. 随机散布（Random Scatter）

**特征**：不良率在一个相对稳定的均值附近随机波动，无明显趋势或模式。

**数据形状**：
```
不良率
  |  █ █  █ █   █ █  █ █
  |__█_█__█_█___█_█__█_█__ 时间
     (围绕均线随机波动)
```

**指向根因**：可能是**系统噪声水平正常**（不良率在统计控制范围内，本次投诉是正常波动中的一次极端事件），也可能是**测量系统不稳定**（量具 GR&R 太高，把测量噪声当成了产品不良）。

**Agent 追问**：
- "这个不良率在 SPC 控制图里是否在控制限内？如果在，这次客户投诉可能是统计上的小概率事件，不是过程恶化。"
- "测量系统的 GR&R 是多少？有没有可能不良率波动其实是测量噪声？"

## Agent 使用方式

1. D2-q4 收集到按时间分布的数据后，让用户画一个简单的时间序列（可以是列表、可以是草图）。
2. 对照以上六种模式判断最接近哪一种。
3. 把模式判断写进 D2 交付物，模式指向的根因方向作为 D4 S1 的优先探索方向。
4. 如果用户说"没有按时间的数据"，这本身就是一个 Insufficient 信号——D2 必须收集时间分布。
