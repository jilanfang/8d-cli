# D4 贝叶斯假设追踪

> Agent：在 D4 S1（发散探索）后，初始化一个候选根因概率表。在每轮 S2（收敛）和 S3（反事实）后，更新概率。根据概率分布决定下一步探索方向。

## 为什么需要概率追踪

人类在根因分析中最致命的认知偏误是"锚定效应"——找到一个看似合理的根因后就锁定它，后续所有证据都被用来"证明这个根因对"，而不是"客观评估哪个根因最可能"。

概率表不是精确的数学计算（因为概率是主观估计的），它的价值在于强迫 Agent 和用户做三件事：
1. 始终保留多个假设，不 prematurely 收敛到单一答案
2. 每收到新证据时，更新所有假设的概率——不只是你最喜欢的那个
3. 当某个假设概率降到很低时，明确标记为"排除"而非让它模糊地留在背景里

## 初始化规则

D4 S1（发散探索）完成后，Agent 初始化一个概率表：

```
候选根因           | 初始概率 | 关键证据                     | 最可能推翻该假设的证据
-------------------|---------|-----------------------------|------------------------
原因A: [描述]      |  35%    | [支持证据1, 2]               | [如果X不成立，则A不成立]
原因B: [描述]      |  30%    | [支持证据1]                  | [如果Y成立，则B不成立]
原因C: [描述]      |  20%    | [支持证据1]                  | [等待验证]
系统交互(A+B)      |  10%    | [理论可能]                   | [如果A或B被排除]
其他（未识别原因）   |   5%    | —                           | —
```

**初始化原则**：
- 基于 D2 的 Is/Is-Not 和时间模式，给最符合的模式更高的初始概率
- 总概率 ≤ 100%（保留"其他未知原因"的份额，通常 5-15%）
- 每个假设必须写一个"最可能推翻它的证据"——如果写不出来，说明这个假设不可证伪，是伪假设
- 初始概率不是精确计算，是基于 D2 证据的合理估计

## 更新规则

每完成一轮 S2（收敛追问）或 S3（反事实挑战），Agent 更新概率表：

### 概率上升的情况
- 新证据直接支持该假设（如：切片确认了假设的物理机制）
- 反事实挑战通过（排除了一个竞争假设，其他假设概率自然上升）
- 该假设能解释多个独立观察的事实（解释力强）

### 概率下降的情况
- 新证据和该假设矛盾
- 反事实挑战失败（"如果X是原因，那Y应该也出现但Y没出现"）
- 该假设不能解释新发现的事实（解释力弱）
- 找到了更强的竞争假设

### 排除规则
- 概率降到 < 5% → 标记为"暂排除"，附排除理由
- 用户和 Agent 都同意某假设不成立 → 标记为"排除"
- 但不删除——万一后续证据翻案，排除记录可以追溯

### 更新幅度
- 主观概率，不需要精确计算
- 但必须回答"为什么调高了/调低了"，不能凭空改数字
- 如果 Agent 不确定调整幅度，直接问用户："你看了这个新证据后，对A的信念是变强了还是变弱了？强/弱多少？"

## 探索方向决策

根据当前概率分布，Agent 决定下一步 S1 的探索方向：

| 概率分布状态 | 下一步动作 |
|------------|---------|
| 最高概率 < 40% | 继续 S1 发散——还没有聚焦的基础，可能遗漏了重要方向 |
| 最高概率 40-70% | 进入 S2 深度收敛——对领先假设深追物理机制，同时对第二假设做一轮反事实 |
| 最高概率 > 70% 但反事实未过 | S3 反事实——集中火力挑战领先假设，同时给第二假设做一次"排除验证" |
| 最高概率 > 80% 且反事实通过 | 可以进入 S5——但必须声明剩余不确定性（"其他"的概率） |
| 两个假设概率相近（差距 < 15%） | 系统交互可能性上升——追问"是否可能两个原因共同作用" |

## 概率表写进 D4 交付物

D4 S5 交付物包含最终的概率表（或至少前三名候选根因及其概率）：

```markdown
## 根因概率评估

| 根因假设 | 最终概率 | 支持证据 | 排除/降低的关键证据 |
|---------|---------|---------|-------------------|
| [根因A] | 75% | [证据1,2,3] | — |
| [根因B] | 15% | [证据1] | [反事实证据X] |
| [系统交互] | 5% | — | A独立解释大部分证据 |
| 其他未知 | 5% | — | 不确定性：长时间可靠性数据尚未充分收集 |

**剩余不确定性**：[A虽然概率最高，但D6验证只有1000台，长时间可靠性数据还需要3个月跟踪。]
```

## Agent 使用方式

1. D4 S1 完成后，初始化概率表（口头向用户呈现，不需要精确）。
2. S2/S3 每轮循环后，更新概率表并说明原因。
3. 根据概率分布决定下一步方向。
4. S5 把最终概率表写进 D4 交付物。
